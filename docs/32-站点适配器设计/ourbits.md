## 编码矩阵规则

| 编码源 | H.264 | HEVC（含 HDR 信息） |
|--------|-------|---------------------|
| FHD Blu-ray | 720p, 1080p | 1080p |
| UHD Blu-ray | × | 1080p, 2160p |

- 移动视频仅接受 720p
- 不接受非 Blu-ray 源的重编码

## 允许的资源类型

1. 完整高清媒介：蓝光原盘（3D 原盘必须是 ISO 格式）、HD DVD 原盘等
2. OurBits 小组 remux：
   - 必须使用 eac3to demux，须包含 eac3to log 信息
   - 必须封装为 MKV 格式
   - 必须包含源语言、英语（如果有）和中文（如果有）的音轨和字幕
   - 须包含章节信息（如果有）
   - 鼓励添加原创字幕、音轨
3. 来源于上述媒介的高清重编码（须符合编码矩阵）
4. HDTV 流媒体
5. 视频网站下载的高清流媒体
6. 标清视频（仅在无高清片源时允许 DVD 原盘）

## 禁止的资源类型

- FRDS 小组、FGT 小组的资源
- 非官方小组 remux 资源
- REMUX、WEBrip 文件
- 电子书、音乐、写真、软件等非影视资源
- DVDRip、CNDVDRip
- 标清 upscale 视频文件
- RealVideo/RMVB/RM/flv 文件
- 单独样片（须与正片一起上传）
- RAR 等压缩文件
- 损坏文件、垃圾文件
- 非 Blu-ray 源的重编码
- 带水印视频
- 带非中英硬字幕的资源（蓝光原盘/remux 除外）
- 改档资源（添加字幕、修改文件夹结构、修改文件名、增删文件等）
- Adult Video（明确禁止）

## Dupe 重复判定规则

官方小组在 Dupe 规则中具有优先权。官方组已发布后，Dupe 规则终止。

### 官方组未发布时的规则：

| 类型 | 规则 |
|------|------|
| 单集 | 剧集完结后不允许再发任何版本单集 |
| WEB-DL | 不允许添加非 WEB 原生素材，相同质量只保留一个 |
| Remux | 相同质量只保留一个，不含中英字幕的不允许发布 |
| 录制 | 不同分辨率只保留一个版本 |
| DVD | 仅无高清片源时保留，只保留一个 |
| Encode | 只允许 Scene 与 P2P 组各一个版本，优先保留内封中文字幕版本 |
| 蓝光原盘 | 不参与 Dupe，但相同版本后发布的会被删除 |
| 蓝光 DIY | 只保留一个版本，不含中文字幕的不允许发布 |

不同版本（加长版/影院版/家庭版/CC 版/4K 重制版等）不互相 Dupe。

## 标题格式规范

### 电影/演唱会/动漫/纪录片

```
名称.[剪辑版本].[年份].[发布说明].来源.分辨率.视频编码.[音频编码]-发布组名称
```

示例：`The Magnificent Nine 2016 720p Blu-ray x264 DTS-OurBits`

### 电视剧/综艺/动漫

```
名称.S**E**.[年份].[发布说明].分辨率.来源.视频编码.[音频]-发布组名称
```

示例：`Always With You 2018 S01E23 1080p HDTV H.264 AAC-iLoveTV`

### 体育

```
名称.年年年年-月月-日日.客场队 vs 主场队.分辨率.来源.视频编码.[音频]-发布组名称
```

示例：`NBA 20181105 Lakers vs Raptors 1080p HDTV H264 AAC-iLoveTV`

### 原盘标题

原盘标题需注明版本信息，如 `Criterion Collection`、`EUR` 等。

## 副标题格式规范

- 中文开头必须是资源的中文名
- 不要使用全角标点
- 音轨/字幕分别放入中括号
- 禁止博取眼球、主观色彩、剧透的字眼

### 电影格式

```
中文片名 (剪辑版本) | 有关影片的有价值信息 [音轨信息] [字幕信息] 发布者附言
```

### 剧集格式

```
中文名 第X季 第X集
```

### 体育格式

```
中文名 比赛双方（严禁透露比赛结果）
```

## 打包规则

仅允许以下资源打包：

1. 按套装售卖的高清电影合集
2. 整季电视剧/综艺/动漫（**禁止多季混合打包**）
3. 同一专题的纪录片
4. 分卷发售的动漫剧集、角色歌、广播剧等

打包要求：
- 必须来源于相同类型媒介
- 必须有相同分辨率水平
- 音视频编码格式一致
- 属于相同发布组（预告片例外）

不符合打包规则的资源直接删除。

## 简介/截图要求

### 通用要求

- 所有影视资源必须包含三张或以上**原始截图**
- 尽可能包含文件详细信息（格式、时长、编码、码率、分辨率、语言、字幕等）

### Info 要求

- 蓝光原盘须包含 **BDInfo** 信息（非 MediaInfo）
- 其他资源须包含 **MediaInfo** 信息
- DVD 原盘须包含一个 vob 及一个 ifo 的信息
- 可以使用解析后的 info，但仍须提供完整原始 info

### 体育特殊

- 严禁在文字介绍/截图/文件名/文件大小/时长中泄露比赛结果

## H&R 规则

- H&R 作为种子待遇出现，仅对大体积资源生效
- 对 **VIP 之外的所有会员**生效
- 对发布超过 **30 天**的种子不生效
- 下载量不足种子总体积 **10%** 不激活 H&R
- 激活后 **14 天内**必须做种 **48 小时**或分享率达到 **1.0**
- 每次违规获得一个 HR 警告，累计 **10 次**封号
- 消除一个 HR 警告需 **10000 魔力值**
- VIP 等级不受此规则约束

## 账号保留规则

| 等级 | 规则 |
|------|------|
| Veteran User 及以上 | 永远保留 |
| Elite User 及以上 | 封存账号后不会被禁用 |
| 封存账号 | 连续 400 天不登录禁用 |
| 未封存账号 | 连续 150 天不登录禁用 |
| 无流量用户 | 注册 7 天后禁用 |

分享率低于 0.3（下载量 > 50GB）禁用下载权限，下载量 > 100GB 直接禁用账号。

- 发布 **72 小时内**盒子上传最多获种子大小 **3 倍**上传量
- 发布 72 小时后正常计算
- 盒子下载量按优惠等级加成：

| 种子优惠 | 盒子实际下载 |
|----------|-------------|
| Free/30% | 50% |
| 50% | 100% |
| 100% | 150% |

- 发布者始终双倍上传，不受盒子规则限制
- VIP 不受盒子规则限制

## 特殊说明

1. **processing_sel = 地区**：字段名为处理方式，实际值为中国大陆/欧美/港台/日/韩/其他
2. **媒介区分 FHD/UHD**：FHD Blu-ray(1) 与 UHD Blu-ray(12) 独立，含 UHDTV(13)
3. **标签双系统**：候选表单用 checkbox name（tagDIY/tagSF 等），实际 upload 用 tags[] 字符串值（diy/sf 等）
4. **11 个标签含 HDR 细分**：杜比视界(db)/菁彩HDR(hdrvivid)/HDR10(hdr)/HDR10+(hdrp)/HLG(hlg)
5. **制作组仅"原创/原抓"**：无外部制作组下拉列表，官方组通过标题后缀标识
6. **禁止 REMUX**：非官方组 remux 禁止发布，官方组 remux 须遵循严格规范
7. **3D 电影独立分类**：Movies-3D(402)，3D 原盘必须是 ISO 格式
8. **电视剧分集/合集拆分**：TV-Episode(412) 和 TV-Pack(405) 独立分类
9. **IMDb 和豆瓣必填**：有了信息展示后不用手动填简介
10. **截图必须原始截图**：至少 3 张，拒绝 QQ 截图，须用播放器截图
11. **原盘须 BDInfo**：非 MediaInfo
12. **禁止转载 HosT 种子到空耳 PT**
13. **禁止更改/去除制作组后缀**
14. **禁止 Adult Video**
15. **禁止改档资源**：自行添加字幕、修改文件夹结构、修改文件名、增删文件等
16. **发布者双倍上传量**：不受盒子规则限制
