## 一、站点角色与适配器策略

### 1.1 仅源站

KeepFRDS 是 **全站官种站**，不接受其他站点转入。upload.php 对普通用户返回：

> "你没有发布种子的权限。如果你是普通会员，请到候选区提交你的发布请求"

**只有上传员及以上等级的用户才能自由上传。**

### 1.2 适配器策略

- **角色**：仅配置为源站（Source）
- **RSS 消费**：可以抓取种子列表 RSS，用于转发到其他站点
- **禁止发布**：不能作为目标站使用
- **转载规则**：转发 FRDS 资源须在出种 24 小时后，并注明出处

### 1.3 上传限速

| 用户等级 | 最高上传速度 |
|---------|------------|
| Extreme User | ≤ 12.5 MB/s |
| UU | ≤ 25 MB/s |
| NM 及以上 | 不限制 |

## 三、关键适配器设计要点

### 3.1 仅源站

KeepFRDS 不能作为目标站，适配器需在站点配置中标记为 **仅源站（SourceOnly=true）**。

### 3.2 转载规则

转发 FRDS 资源到其他站点时：
- 须在出种 **24 小时后** 才能转发
- 必须 **注明出处**（KeepFRDS/FRDS）

### 3.3 Cloudflare 防护

站点使用 Cloudflare（cookie 含 `cf_clearance`），需参考 `docs/31-模块设计决策记录.md §29` 的 TLS 指纹绕过方案。

### 3.4 HEVC 编码细分

KeepFRDS 的 HEVC 编码极度细分（5 个子类），适配器在解析 RSS 时需根据标题/MediaInfo 精确识别 HDR10/HDR10+/Dolby Vision/10bit。

### 3.5 黑名单制作组

FRDS 出现在 SpringSunday 的黑名单中。适配器转发链路需检查目标站是否接受 FRDS 来源。

### 3.6 分类特殊值

- 419（🍆）为成人内容，与萝莉的 9KG、PTFans 的特别区类似
- 425（鲨鱼）为站点特色分类，需确认具体内容类型

### 3.7 自定义前端

KeepFRDS 使用 Tailwind CSS 深度定制前端，非标准 NexusPHP 样式。HTML 结构可能需要额外的解析逻辑。

## 五、站点规则（rules.php）

> 以下内容来源于 2026-04-23 采集的 rules.php 页面（需 Cookie 认证，5 个 cookie：c_secure_uid/c_secure_pass/c_secure_ssl/c_secure_tracker_ssl/c_secure_login）。

### 5.1 账号保留规则

| 条件 | 期限 |
|------|------|
| Veteran User 及以上 **或注册满 5 年** | 永远保留 |
| Elite User 及以上封存账号 | 允许封存 |
| 封存账号连续不登录 | **90 天**封禁 |
| 未封存账号连续不登录 | **35 天**封禁 |
| 无流量用户连续不登录 | **7 天**删除 |

**封禁恢复机制**：封禁后下次登录尝试扣除 1 个邀请或 `2400B × 逾期天数` 魔力值重新激活。

### 5.2 总则特有规则

- **转载规则**：PT@KEEPFRDS 资源转载请在出种 **24 小时后**
- **网盘禁止**：禁止将本站资源上传至网盘（百度盘、Google Drive 等）及 BT 站点后公开分享（受众 >3 人）
- **隐私保护**：禁止在任何公开的社交媒体、论坛、群讨论任何与本站相关的话题
- **多 IP 限制**：Tracker 支持单用户多 IP 同时下载，但严禁多开刷流及免费种做种时反复/大量下载刷流

### 5.3 下载规则 — 种子促销

#### 5.3.1 新种递减促销（时间驱动）

| 阶段 | 条件 | 促销类型 |
|------|------|----------|
| 发布时 | 所有新种子 | **免费** |
| 发布时 | 总体积 >20GB | **免费 & 2x上传** |
| 发布 1 天后 | 原"免费" | → **50%下载** |
| 发布 1 天后 | 原"免费 & 2x上传" | → **50%下载 & 2x上传** |
| 发布 7 天后 | 原"50%下载" | → **普通** |
| 发布 7 天后 | 原"50%下载 & 2x上传" | → **普通** |

#### 5.3.2 自动促销（做种人数驱动）

| 条件 | 促销类型 |
|------|----------|
| 做种人数 ≤ 3 | 免费 & 2x上传 |
| 做种人数 ≤ 10 | 50%下载 & 2x上传 |
| 做种人数 ≤ 20 | 2x上传 |
| 做种人数 > 20 | 普通 |

### 5.4 上传规则

#### 5.4.1 上传总则

- 发布者获得 **双倍上传量**
- 上传者最高记速：Extreme User ≤12.5MB/s，UU ≤25MB/s，NM 及以上不限制

#### 5.4.2 上传者资格

**只有上传员及以上等级的用户，或者是管理组特别指定的用户，才能自由上传。**

#### 5.4.3 允许的资源规格（FRDS 特色）

| 规格 | 说明 |
|------|------|
| DVDRip | 来源于 DVD 的重编码 |
| MNHD 1080p | 来源于高清媒介的重编码（至少 1080p） |
| MNHD 4k | 来源于高清媒介的重编码（至少 2160p） |
| mUHD 4k HDR | 来源于高清媒介的重编码（至少 2160p），**必须为 HDR 作品** |

**注意**：FRDS 不接受 Blu-ray 原盘、Remux 等非重编码资源，只接受自制重编码（MNHD/mUHD 系列）。

#### 5.4.4 不允许的资源

与标准 NexusPHP 模板一致（upscale/低质量标清/RMVB/有损音频/压缩包等），另加：
- 不允许 **Blu-ray 原盘** 和 **Remux**（只接受自制重编码）
- 不允许非 FRDS 官方制作组发布的资源

### 5.5 Dupe 判定规则

标准 NexusPHP dupe 规则，但**没有**"断种 45 天或 18 个月"的豁免条款（原文中此条缺失）。

### 5.6 资源打包规则

标准 NexusPHP 打包规则模板，与 LemonHD/Farmm 一致。

### 5.7 种子信息标题规范

标准 NexusPHP 标题模板（电影/电视剧/音轨/游戏）。

### 5.8 管理组成员退休待遇

## 六、禁转/限转标记系统（关键规则）

> 以下内容来源于用户提供的详情页、列表页、搜索页观察结果。

### 6.1 转载基本规则

- 转发 FRDS 资源须在 **出种 24 小时后**
- 转发时必须 **注明出处**（KeepFRDS/FRDS）
- 全站官种，**不允许普通用户转载发布**，只能做源站往其它站转发

### 6.2 禁转/限转标记

KeepFRDS 在种子详情页和列表页中通过文字标记区分不同转载状态：

| 标记文字 | 显示位置 | 含义 |
|----------|----------|------|
| `[限转资源]` | 详情页内文字标识 | 限转资源，有条件转载 |
| `[限时禁转]` | 种子列表页种子名称后 | 限时禁止转载，需等待解除 |
| `[禁转]` | 种子列表页种子名称后 | 禁止转载 |
| `【暂时禁转，待确认】` | 种子列表页种子名称后 | 暂时禁转，状态待确认 |

### 6.3 适配器转发过滤规则

转发 FRDS 资源时，适配器必须：

1. **检测禁转标记**：解析详情页/列表页中的以下标记
   - `[禁转]` — **禁止转发**
   - `[限时禁转]` — **禁止转发**（直到标记解除）
   - `【暂时禁转，待确认】` — **禁止转发**
2. **检测限转标记**：`[限转资源]` — 需确认限转条件后决定是否转发
3. **时间检查**：确保出种已满 24 小时
4. **出处标注**：转发时在简介中注明来源为 KeepFRDS/FRDS

### 6.4 全局转载策略影响（§30.5）

- FRDS 的 `[禁转]` 标记与 §30.5 第 2 条"禁止发布源站带禁转/独占/谢绝转载/限时禁转标签"完全对齐
- FRDS 制作组出现在 SpringSunday 黑名单中，转发到 SpringSunday 时需拦截
- FRDS 全站官种特性意味着：所有 FRDS 种子均可视为"有出处"的官种资源
