# PT-Forward 编码规范

> 本文档从设计文档（docs/31）中提取所有编码约定，作为编码阶段的唯一规范参考。
> 设计文档中分散在 §27.1/§33.2/§33.1.70/§33.1.98 等处的规范统一汇总于此。

## 1. 目录结构

```
cmd/pt-forward/          # 程序入口（main.go）
internal/
├── model/               # §33.1 canonical struct + §33.2 canonical interface（纯定义，零业务逻辑）
├── config/              # Config 加载 + Validate
├── adapter/             # 站点适配器实现（NexusPHP/Unit3D/Gazelle/MTeam/TNode/Generic/Custom）
├── client/              # 下载器客户端实现（qBittorrent/Transmission）
├── rss/                 # RSS 引擎
├── seeding/             # 刷流引擎
├── reseed/              # 辅种引擎
├── publish/             # 发布流水线
├── site/                # 站点管理 + CookieCloud
├── notification/        # 通知系统
├── api/                 # HTTP handler + 路由
├── middleware/           # JWT/CORS/限流/错误恢复/日志中间件
├── scheduler/           # 定时任务（TaskRegistry）
├── auth/                # JWT 认证
├── event/               # ConfigEventBus + WebSocket Hub
├── health/              # 健康检查 /healthz
├── util/                # 工具函数（NormalizeTitle/凭据加密/日志脱敏）
configs/                 # 默认 YAML 配置 + 站点默认值
```

**规则**：
- `internal/model/` 只放 struct 定义和 interface 定义，禁止放业务逻辑
- 每个 struct/interface 的字段签名必须与 `docs/31` §33.1/§33.2 canonical 完全一致
- 新增 struct 必须先在 `docs/31` 登记 canonical，再写代码

## 2. 命名规范

| 类别 | 规则 | 示例 |
|------|------|------|
| 包名 | 全小写，单个单词，无下划线 | `seeding`, `reseed`, `publish` |
| 文件名 | 全小写，下划线分隔 | `torrent_event.go`, `site_adapter.go` |
| struct | PascalCase | `SeedingTorrentRecord`, `ContentFingerprint` |
| 接口 | PascalCase + er/Provider 后缀（如已定义则不改） | `SiteAdapter`, `DownloaderProvider` |
| 枚举 | PascalCase 类型 + `const` 块 | `type DiscountLevel string` |
| 方法 | PascalCase，与 §33.2 canonical 签名一致 | `GetTorrentByHash(ctx, hash)` |
| 私有字段 | camelCase | `maxRetries`, `cooldown` |
| 常量 | PascalCase（枚举值）或 SCREAMING_SNAKE（全局常量） | `PriorityPublish`, `ErrNotSupported` |
| DB 表名 | 复数蛇形（TableName() 返回值） | `"seeding_torrent_records"` |

## 3. Context 传递

**所有跨函数调用必须传递 `ctx context.Context`**，作为第一个参数。

```go
// ✅ 正确
func (s *SeedingEngine) Process(ctx context.Context, event *TorrentEvent) error

// ❌ 禁止
func (s *SeedingEngine) Process(event *TorrentEvent) error
```

**§33.2 canonical 签名为准**：其他章节的伪代码可能省略 ctx，编码时以 §33.2 为准。

## 4. 错误处理

### 4.1 AppError 统一错误

```go
// 底层（Repository/Adapter）：返回 AppError 或 fmt.Errorf("%w", appErr)
// 中层（Service/Engine）：按需包装为 AppError（如 NewAppError(14002, "...", err)）
// API 层：handler 返回 error → 全局中间件统一转为信封格式
```

**禁止**在 Service 层使用裸 `errors.New()` —— 必须使用 `AppError` 携带 Code。

### 4.2 Provider Get 方法

Provider 的 Get 方法返回 `(nil, ErrNotFound)` 而非 `(nil, nil)`：
```go
func (r *SiteRepository) GetByID(ctx context.Context, id uint) (*Site, error) {
    var site Site
    err := r.db.First(&site, id).Error
    if errors.Is(err, gorm.ErrRecordNotFound) {
        return nil, ErrNotFound  // ✅ 不返回 (nil, nil)
    }
    return &site, err
}
```

### 4.3 所有 error 必须检查

```go
// ❌ 禁止
_ = someOperation()

// ✅ 必须检查
if err := someOperation(); err != nil {
    return fmt.Errorf("some operation: %w", err)
}
```

## 5. 并发安全（§33.2 全局约定 #348）

| 类型 | 规则 | 接收者 |
|------|------|--------|
| GORM 模型 struct（§33.1.x 纯数据模型） | 无 `sync.*`/`atomic.*` 字段，值对象，每次 DB 读取获得独立副本 | 值接收者 |
| 运行时 struct（含 sync.Mutex 等） | 必须指针接收者，禁止值复制（`go vet` copylocks） | 指针接收者 |
| Provider/Repository 接口实现 | 必须并发安全，多个 goroutine 可同时调用同一实例 | - |
| SiteAdapter（§33.2.3）实现 | 必须并发安全，内部状态用 `sync.Mutex` 保护 | - |

**初始化规范**：运行时 struct 的字段赋值必须在 `Start()` 之前完成；`Start()` 后字段应视为不可变。

## 6. *time.Time nil 防御

全系统多处 `*time.Time` 指针字段，解引用前必须 nil 检查：

```go
// ✅ 统一使用辅助函数
func TimeSince(t *time.Time) time.Duration {
    if t == nil { return 0 }
    return time.Since(*t)
}

// ❌ 禁止（nil 时 panic）
time.Since(*ptr)
```

## 7. 日志规范

### 7.1 日志库

使用 `go.uber.org/zap`，禁止 `fmt.Println` / `log.Println`。

### 7.2 凭据脱敏（§33.1.98）

全局 Zap Logger 注入 `SanitizerCore` 装饰器，自动脱敏以下字段：
- `passkey=xxx` → `passkey=***`
- `cookie=xxx` → `cookie=***`
- `api_key=xxx` → `api_key=***`
- `bearer_token=xxx` → `bearer_token=***`
- `password=xxx` → `password=***`
- `encryption_key=xxx` → `encryption_key=***`

**禁止**在日志中直接打印凭据值：
```go
// ❌ 禁止
logger.Info("login", zap.String("cookie", site.Cookie))

// ✅ 不打印，或脱敏后打印
logger.Info("login", zap.String("domain", site.Domain))
```

### 7.3 结构化日志

```go
// ✅ 使用 zap 的结构化字段
logger.Info("torrent deleted",
    zap.String("client", clientID),
    zap.String("info_hash", infoHash),
    zap.String("reason", reason),
)

// ❌ 禁止字符串拼接
logger.Info(fmt.Sprintf("torrent %s deleted from %s", infoHash, clientID))
```

## 8. SQL 注入防护（§33.1.98）

GORM 参数化查询天然防注入。**禁止**使用 `db.Raw()` + `fmt.Sprintf()` 拼接用户输入：

```go
// ❌ 禁止
db.Raw(fmt.Sprintf("SELECT * FROM sites WHERE domain = '%s'", userInput))

// ✅ 必须参数化
db.Raw("SELECT * FROM sites WHERE domain = ?", userInput)
```

## 9. GORM 使用规范

### 9.1 写操作走 DBWriteQueue

所有 DB 写操作（Create/Update/Delete）通过 `DBWriteQueue.Execute()` 提交：

```go
err := writeQueue.Execute(ctx, func(db *gorm.DB) error {
    return db.Create(&TorrentEvent{...}).Error
})
```

读操作直接用 `db` 实例，不走队列。

### 9.2 深拷贝 SiteConfig

`GetSiteConfig()` 每次返回深拷贝新实例。**禁止**缓存返回值并跨 goroutine 共享可变引用：
```go
// ❌ 禁止
config := provider.GetSiteConfig(ctx, domain) // 获得引用
go func() { useConfig(config) }()             // 跨 goroutine 共享

// ✅ 每次读取
go func() {
    config, err := provider.GetSiteConfig(ctx, domain)
    useConfig(config)
}()
```

### 9.3 API 禁止暴露 GORM 主键 ID

API 返回体中不包含 `id` 字段（uint PK），用 domain/name 等 business key 标识资源。

## 10. Discount 枚举使用（Sprint 81 修复）

```go
// ❌ 禁止硬编码
event.IsFree = true
event.FreeLevel = "FREE"

// ✅ 必须用 Discount 枚举方法
event.IsFree = event.Discount.IsFree()
event.FreeLevel = string(event.Discount)
```

## 11. 凭据加密（§23.3）

Site 模型的 Cookie/Passkey/APIKey 等凭据字段通过 `CredentialEncryptor` BeforeSave/AfterFind 自动加解密：
- `BeforeSave`：明文 → AES-256-GCM 密文
- `AfterFind`：密文 → 明文

编码时只需正常读写字段值，加解密由 GORM hook 自动处理。

## 12. 注释规范

- **禁止添加无意义注释**（如 `// SetName sets the name`）
- **接口方法必须有 godoc 注释**，说明行为和返回值
- **复杂逻辑必须有注释**，说明"为什么"而非"做了什么"
- **引用设计文档章节**：`// §33.1.87 OperationAuditLog`

## 13. 测试规范（§33.1.99）

- 单元测试文件与源文件同目录：`seeding.go` → `seeding_test.go`
- 集成测试用 SQLite 内存 DB：`file::memory:?cache=shared`
- 适配器测试用 `httptest.Server` 模拟 PT 站，不访问真实站点
- CI 启用 `-race` 竞态检测
- 不 mock GORM，直接用内存 DB

## 14. 安全规范（§33.1.98）

- API 响应 CSP 响应头：`Content-Security-Policy: default-src 'none'; ...`
- API 响应安全头：`X-Content-Type-Options: nosniff` + `X-Frame-Options: DENY` + `Referrer-Policy: no-referrer`
- JWT 存 localStorage（非 Cookie）
- 健康检查端点 `/healthz` 无需认证
