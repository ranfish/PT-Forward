## 三、与其他站点对比

### 3.1 HDFans 特殊之处

| 维度 | HDFans | 典型 NexusPHP（如 GTK/13City） |
|------|--------|-------------------------------|
| 媒介细分 | 20种（UHD原盘/DIY/Remux/压制分开） | 10-13种（简单列表） |
| 视频编码 | 区分 H.264 和 x264、H.265 和 x265 | 合并为 H.264 或 x264 |
| 音频编码 | 24种（含 DSD、ALAC、TTA 等） | 通常无此字段 |
| 地区字段 | 有（13个选项） | 通常无 |
| 制作组 | 30个（含 PTP、BTN、GGN 等外站组） | 10-12个（以国内组为主） |
| 标签数 | 27个（含 AI修复、刮削、Hi-Res 等） | 6-7个 |
| 分类数 | 16个（含戏剧、演唱会等） | 8-12个 |

### 3.2 关键差异

1. **媒介按分辨率+类型细分** — 不是简单的 "Encode"，而是 "UHD压制"（20）vs "1080P/i压制"（24）vs "720P压制"（25），需要根据分辨率+是否DIY动态选择
2. **编码区分原盘和压制** — H.264/AVC（1）vs x264（2），H.265/HEVC（3）vs x265（4），需要根据媒介类型选择
3. **源站转发标签** — 需同时满足"源站匹配"和"官方种子"两个条件

## 五、Hook 实现要点

### 5.1 媒介细分规则

HDFans 的媒介需要根据分辨率+是否DIY动态选择，不能简单映射：

```
标准媒介 → HDFans 媒介映射逻辑：

UHD + 原盘 + DIY   → 18 (UHD DIY)
UHD + 原盘 + 原生   → 17 (UHD原盘)
UHD + Remux         → 19 (UHD Remux)
UHD + Encode        → 20 (UHD压制)
BD  + 原盘 + DIY    → 22 (BD DIY)
BD  + 原盘 + 原生    → 21 (BD原盘)
BD  + Remux         → 23 (BD Remux)
Encode + 1080p/i    → 24 (1080P/i压制)
Encode + 720p       → 25 (720P压制)
Encode + SD         → 26 (MiniSD)
WEB-DL              → 5
HDTV                → 6
DVD                 → 7
CD                  → 9
```

### 5.2 编码细分规则

HDFans 区分原盘编码和压制编码：

```
原盘/Remux → H.264/AVC(1), H.265/HEVC(3)
压制/Encode → x264(2), x265(4)
```

需根据媒介类型选择正确的编码值。

### 5.3 源站转发标签条件

添加"源站转发"标签需同时满足：
1. 种子来自某个站点的官方发布
2. 转发的是该站点自己的官方种子（不是第三方在该站发布的种子）

例如：从 CHD 转发 CHD 官方种子 → 可以用"源站转发"；从 CHD 转发 HDC 官方种子 → 不能用。

### 5.4 双语标签增强

当种子同时有"国语/粤语"标签和"中英双语字幕"时，自动添加"中英双语"标签。

## 六、完整站点规则（rules.php + 论坛公告）

> 数据来源：rules.php（9342 字符）+ topicid=2662（H&R V2.0）+ topicid=2（发布注意事项）+ forumid=7&topicid=27（发种图文教程）

### 6.1 账号保留规则

| 等级/状态 | 保留条件 |
|-----------|----------|
| Extreme User 及以上 | 永远保留 |
| Veteran User 及以上（封存） | 封存后不会被删除 |
| 封存账号 | 连续 180 天不登录删除 |
| 未封存账号 | 连续 90 天不登录删除 |
| 无流量用户 | 连续 15 天不登录删除 |

### 6.2 促销规则

**随机促销概率**：

| 概率 | 促销类型 |
|------|----------|
| 10% | 50% 下载 |
| 5% | 免费 |
| 5% | 2x 上传 |
| 3% | 50% 下载 & 2x 上传 |
| 1% | 免费 & 2x 上传 |

**自动免费**：文件总体积 >20GB、Blu-ray/HD DVD 原盘、电视剧每季第一集

**促销时限**：除"2x 上传"无时限外，其余限时 **3 天**

### 6.3 上传总则

- 做种时间不足 24h 或故意低速上传将被警告甚至取消上传权限
- **发布者获得双倍上传量**
- 游戏类资源仅上传员及以上可自由上传，其他用户须候选

### 6.4 Dupe 判定规则

**来源优先级**：Blu-ray/HD DVD > HDTV > DVD > TV

- 同一视频高清版使标清版被判定为 Dupe
- 动漫类 HDTV 和 DVD 同优先级（特例）
- 相同媒介+相同分辨率：按发布组优先级（参考 Scene & Internal 规范）
- 保留一个 DVD5 大小（~4.38GB）最佳画质重编码版本
- 基于无损截图对比，高质量使低质量被判定为 Dupe
- 不同区域/不同配音/字幕的原盘不被视为 Dupe
- **每个无损音轨原则上只保留一个版本（分轨 FLAC 优先级最高）**
- 旧版连续断种 45 日或发布 18 个月以上，新版不受 Dupe 约束

### 6.5 标题命名规范

**电影/演唱会**：`英文或拼音名称.剪辑版本.年份.发布说明.媒介.分辨率.视频编码.音频编码-制作组`

**电视剧/纪录片**：`英文或拼音名称.S**E**.年份.发布说明.分辨率.来源.视频编码.音频-制作组`

**副标题**：中文片名开头，使用半角标点。音轨/字幕信息放中括号，发布者留言放 `*……*` 内。转载不用写转自某某站，在简介中感谢源站即可。

### 6.6 简介格式（标准模板）

1. **海报**：全尺寸海报上传图床（推荐 imagelol.com / z4a.net / imgbox.com）
2. **简介信息**：导演/演员/年份/评分等（推荐 PT-Gen 生成）
3. **影片详细信息**：视频/音频编码信息，使用 `[quote][/quote]` 包裹
4. **资源截图**：若干张截图上传图床

### 6.7 来源与媒介选择指南

- 重编码作品来源选"压制"，UHD 原盘/BD 原盘指未经改动（DIY/Remux）的资源
- 音乐资源来源选"抓取"，录制用于 HDTV 资源
- UHD 原盘未改动 → 来源 UHD 原盘 + 媒介 UHD Blu-ray
- UHD 改动(DIY) → 来源 UHD DIY + 媒介 UHD Blu-ray
- UHD 压制 → 来源压制 + 媒介 UHD Blu-ray

## 七、H&R 规则（#2662 V2.0）

> 数据来源：forums.php?topicid=2662（全站 H&R 公告）

### 7.1 全站 H&R（2024-05-20 V2.0 生效）

- **所有种子**都有 H&R 设置（种子区 + 特别区各 10 点 HP）

### 7.2 考核标准

- 考核时间：种子**下载 100%** 后的 **720 小时**（30 天）
- 考核要求：保种 **72 小时** 或单种分享率达到 **1:1**
- 考核时间设置较长，保证不能长期开机的用户也能完成

### 7.3 HP 系统

- 每会员初始 **10 点 HP**

## 九、盒子规则（rules.php）

### 9.1 盒子定义

拥有较大上下行带宽的服务器、VPS 或主机。

### 9.2 速度限制

- 单种上传速度最高 **200Mbps**（约 25MB/s）
- 超过此速度的教育网/家宽用户需自行限速
- 暂不支持教育网/家宽作为盒子登记

### 9.3 黑种

- 盒子不享受种子优惠，按 **100%** 下载量计算

### 9.4 上传限额

- 盒子做种 **72h 内只享受种子体积×3 的上传量**

### 9.5 不被认可的盒子

**羊毛盒**：GCP、AWS、Azure、Oracle、Vultr、Linode、DigitalOcean、Hetzner CX/CPX 系列（Hetzner 独立服务器可以报备）

**共享 IP 盒子**：无独立 IP 的盒子

**其他**：Greencloudvps/绿云、BuyVM、Netcup、liteserver 及未列入的大盘/中盘 VPS

### 9.6 处罚

- 单种上传超过 25MB/s 且未报备：禁用下载权限
- 单种超过 100MB/s：第一次警告，二次禁用账号
- 使用不认可盒子登记：第一次警告，第二次扣 500G 上传（不足则清零），第三次封号
- 提交虚假信息登记：禁用账号
- VPS/代理软件反代盒子洗白家宽 IP：直接禁用账号

### 9.7 豁免

- VIP 用户在 VIP 期间不受盒子限制
- 工作组/管理组不受限制
- 已备案盒子发种建议速度不超过 100MB/s
