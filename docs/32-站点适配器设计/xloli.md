## 一、总则（"没有规则的规则"）

站点采用独特的"没有规则的规则"哲学，总则页面以散文形式描述隐性规则。实际规则分散在多个论坛帖子中。

### 1.1 账号保留规则

1. Veteran User 及以上等级永远保留
2. **Crazy User** 及以上等级封存账号后不会被删除（注意：非标准的 Elite User）
3. 封存账号连续 **400 天**不登录 → 封禁
4. 未封存账号连续 **150 天**不登录 → 封禁
5. 零流量用户连续 **60 天**不登录 → 封禁（注意：60 天，非标准 100 天）

### 1.2 站点特有禁令

- 禁止在公开场合暴露本站网址（可用"爱萝莉"/"萝莉站"/"iloli"替代）
- 禁止使用自动化脚本/程序进行非手动签到
- 禁止使用自动化工具在群聊区喊话
- 第一次捣乱警告，**第三次**封禁（非标准的第二次）
- 禁止代刷上传/下载/魔力值（违者视为重大违规，禁止再次进站）

### 1.3 新手考核

30 天内需达到：
- 上传增量 > **20GB**
- 平均做种时间 > **24 小时**
- 做种积分 > **4500**
- 支持捐赠免考

### 1.4 发布权限

| 区域 | 最低等级 |
|------|---------|
| 综合区（普通区）发种 | **Elite User** |
| 特别区（9KG）发种/查看/下载 | **Power User** 或注册时间 > **30 天**（满足其一即可） |
| 匿名发种 | User |

## 三、上传规则

### 3.1 上传总则

- 做种时间不足 **24 小时**或故意低速上传 → 警告甚至取消上传权限
- 发布者获得 **1.5 倍上传量**
- 违规但有价值的资源 → 告知管理组，可能破例
- **别发种倒垃圾，这是最高原则**

### 3.2 不允许的资源

- 单本电子书资源
- 网课资源
- 软件类体积 < **300MB** 的无分流意义资源（经管理组许可的有价值小软件除外）
- 公网 BT 盗转 PT 的制作组资源（见禁发制作组列表）
- 损坏文件/垃圾文件
- 违反 Dupe 规则的资源

### 3.3 发布分类规则

- **所有动画资源**（包括动画电影和动画电视剧）→ 类型必须选择 **动画 (Animation)**，包含欧美动画电影

### 3.4 分集资源规则

- 分集资源必须从**第一集**开始发布
- 原则上不允许烂尾，需要更新完毕，否则发布权限可能取消
- 例外：源站断更或管理组许可

### 3.5 已完结资源规则

- 已完结的剧集/动漫/综艺/纪录片禁止以分集形式发布
- 已完结动画的 BDRip 已出时，不欢迎格式不规范的 WEB-DL/WEBRip

### 3.6 打包合集规则

- 影视动画资源禁止发布**非同一制作组**的打包合集（极优质/极稀有除外，须管理组许可）
- 禁止自拼电影合集，允许官方电影合集
- 多季度资源：如原发布组未发布合集，禁止自行打包；如原发布组发布了合集，禁止拆分发布
- 除非确有必要，转载资源请勿更改文件夹架构

### 3.7 语言要求

禁止发布无以下任一语言的电视剧/电影/纪录片资源：
- 中文字幕
- 国语配音
- 粤语配音
- 日语配音

**例外**：动画、动画衍生作品、原产地为大陆/港/台/日的资源、无对白视频（勾选无对白标签）、字幕区已上传中文字幕（勾选中字标签）、管理许可

### 3.8 低分电视剧禁发

禁止发布豆瓣评分 ≤ 7.0 的国产/港台电视剧（限定平台）：
- 适用：优酷 HQ / 爱奇艺 EDR / **腾讯臻彩MAX**
- 豆瓣无评分视为 < 7.0
- 目前对电影暂无限制
- 违规处罚：**10 倍上传量**

### 3.9 违规做种

- **严重限速**：上传速度 > 8Mbps（1MB/s）一般不会认定
- **网盘做种**：115/百度网盘/OneDrive/Google Drive 等非本地存储
- 排查：程序自动 + 人工手动
- 处罚：警告或封禁
- 不建议使用 Transmission 保大种子（>1GB），建议 qBittorrent

### 3.10 违规处罚标准

- 扣除上传量 = **min(真实上传量 × 10, 种子体积 × 10)**

## 五、Dupe 判定规则

### 5.1 共存规则

- **不同制作组**作品允许共存
- **不同分辨率**作品允许共存
- **不同编码**作品允许共存
- 同一制作组的同一分辨率**不允许重复**（包括增删字幕/NFO 改变 hash）
- 不同区域/不同配音/字幕的 Blu-ray/HD DVD 原盘不算 Dupe

### 5.2 Web-DL Dupe 特殊规则

对于已充分保种的完结 WEB-DL 资源：相同分辨率、编码甚至同源的 WEB 将被视为 Dupe。种审可删除无意义的新 WEB-DL。

### 5.3 动漫 WEB-DL/WebRip Dupe

- BDRip 已出时，原则上不欢迎格式不规范的 WEB-DL/WEBRip
- 如质量超过 BDRip / 有特殊音轨 / 有必要字幕 → 例外
- 同一制作组的 WEBRip 原则上只接受一个 1080p 版本
- 字幕优先级：繁简日内封 > 简日内封 > 简日内嵌 > 简繁内封 > 简体内封 > 简体内嵌 > 其他

### 5.4 替换规则

新版本允许替换旧版本的条件：
1. 同一制作组新版本修复了旧版本确认的错误/画质问题
2. 新版本来源有更好的质量
3. 旧版本连续断种 **45 日以上**且发布 **6 个月以上** → 允许重发

### 5.5 Web-DL 质量排序（动漫）

- 质量更高的 WEB-DL 存在时，禁止上传质量更低的版本
- 国漫 4K 非HQ > 国漫 4K（出于保种考虑，不包括电影）
- 4K SDR > 4K HDR/DoVi（出于保种考虑，不包括电影；原始片源明确为 HDR 制作的例外）
- 新番期间质量更差但更新更快的源允许共存，完结后禁止

## 六、种子发布规范简版

### 6.1 标题

采用 0day 命名（U2 命名亦可；9KG 资源采用 kamept 命名亦可；游戏用 kamept/pterclub/0day/TJUPT 命名；漫画用 azusa/U2/TJUPT/BYR 命名）

### 6.2 动漫 WEBRip/Web-DL 标题

- 英文名优先采用 **MAL** 的罗马英文名称
- 中文名称建议采用 **BGM（Bangumi）** 的中文名称
- 日文名称采用 BGM 的日文名称
- 副标题可补充其他中文/英文名

### 6.3 副标题

至少要有中文名称。

### 6.4 简介

转载声明/制作说明/致谢 + 海报 + 豆瓣简介 + **3 张 PNG 无损截图**

### 6.5 MediaInfo

- 建议英文版本 MediaInfo（简化版也可但不推荐）
- BD 用 BDInfo 或简化版 BDInfo
- DVD 原盘可填 IFO + VOB 两个 MediaInfo

## 二、关键适配器设计要点

### 2.1 9KG 区

- 适配器使用 `browsecat`（data-mode=4）
- `specialcat`（data-mode=6）含 11 个分类（418-430）

### 2.2 source_sel = 地区

`source_sel` 表示地区（大陆/港台/日韩/欧美/俄国/泰国等），非来源媒介。含泰国和俄国，与其他站不同。

### 2.3 动漫向定位

制作组以动漫 Raw 组和压制组为主（VCB-Studio、AI-Raws、Snow-Raws 等），适配器在匹配制作组时需优先匹配动漫相关组。

### 2.4 两个区域字段完全相同

综合区和 9KG 区的所有质量字段值完全一致（source_sel/medium_sel/codec_sel/audiocodec_sel/standard_sel/team_sel），仅字段名后缀不同。适配器只需使用 `[4]` 后缀。
