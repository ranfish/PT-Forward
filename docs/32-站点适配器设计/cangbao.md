## 一、发布页面表单字段分析

**提交地址**: `takeupload.php`（POST multipart/form-data）

### 1.1 模式系统

使用 `data-mode='4'` 属性控制字段显示，字段名带 `[4]` 后缀。标准综合区模式。

### 1.2 基础信息字段

|--------|----------|------|------|
| `file` | file | ✓ | 种子文件 |
| `name` | text | ✓ | 标题 |
| `small_descr` | text | - | 副标题 |
| `url` | text | - | IMDb 链接 |
| `pt_gen` | text | - | PT-Gen 链接 |
| `nfo` | file | - | NFO 文件 |

## 二、发种规则（论坛规则帖）

### 2.1 总则

- 推荐 4K 及更高分辨率资源
- 优先选择带简体中文字幕的资源
- 低于 1080p 的电影慎重发布（极具收藏价值/稀有/经典老片且站内无更高清版除外）
- 避免滥发低价值资源，避免大量重复发布

### 2.2 剧集规则

- 剧集转载**必须选择完结标签**，严禁分集发布（官方制作组除外）
- 从第一季开始发种，如无法持续更新则不要轻易发布
- 禁止单集发布（审核脚本会检测 S01E01 等模式，非官组触发错误）

### 2.3 命名与元数据

- 严格遵循标准命名规范，包含片名、年份、分辨率、视频编码、音轨、制作组等
- 确保文件完整性，上传前进行完整性校验
- IMDb 链接、影片简介、演员表、截图、MediaInfo 必须准确无误
- 首发与重发原则：优先发布本站尚未收录的资源；重制/更高质量版本需注明原因
- 多季合集命名格式：`[中文译名].[英文原名].[年份].[季].[最高分辨率标识].[来源].[编码].[音轨]-[制作组].[容器]`

### 2.4 做种义务

- 发布者有长期做种义务
- 建议至少保持 1:1 分享率，尽可能延长做种时间

### 2.5 多季合集规范

多季合集（S01-SXX）要求高度一致性，不符合标准将被拒绝或移除：

- **分辨率统一**：所有季度分辨率必须完全一致，禁止 1080p 和 720p 混杂
- **格式/编码统一**：封装格式、视频编码、音频编码必须一致，避免不同压制者资源混搭
- **制作组统一**：应尽量来自同一制作组，中途更换需在画质/编码上高度匹配
- **完整性**：必须包含该系列所有官方发布正片，不得缺集

**合集权限**：
- 普通用户：仅查看/收藏
- UP（上传员）：创建/编辑合集，添加编辑种子，**禁止删除**
- AD（管理员）：最高权限，含删除
- 删除合集内种子需联系管理员审核执行

### 2.6 动画与动漫分类规则

- **动画(404)**：仅限单集作品（电影/单集OVA等）
- **动漫(405)**：多集内容（含分集/合集/完结标识）必须选择动漫分类
- 误选检测：动画分类中检测到多集信号→错误；动漫分类中无多集信号→错误

### 2.7 短剧分类

- 副标题含"短剧/竖屏剧/微短剧/竖屏"关键词时建议选短剧(403)分类

### 2.8 音乐/书籍分类豁免

音乐(409)和书籍(410)分类跳过大部分质量字段检查（媒介/编码/音频/分辨率），仅检查分类是否选择。标签限制为：官方、国语、粤语、中字、英字、分集、完结、合集。

## 四、藏宝阁特殊注意事项

### 4.1 所有质量字段值倒序排列

藏宝阁是已采集站点中**唯一一个所有质量字段值从大到小倒序排列**的站点：
- 媒介：UHD=9 → VCD=1
- 编码：AV1=9 → H.261=1
- 音频：PCM=9 → MP3=1
- 分辨率：4320p=7 → 480P=1

这种设计使得值越大代表质量越高，在排序时自然有序。适配器映射时不能假设值与常见站点相同。

### 4.2 MetaFlow 自动填表

前端集成了 MetaFlow 工具（`js/metaflow.js`），可自动从 PT-Gen 链接提取信息填充表单。这不是表单字段，不影响 POST 数据，但意味着其他站点的种子可能需要重新映射字段。

### 4.3 分类细分

藏宝阁的分类比大多数 NexusPHP 站点更细：
- "动画"(404) 与 "动漫"(405) 是独立分类
- "短剧"(403) 独立于 "剧集"(402)
- "儿童"(406) 和 "书籍"(410) 作为独立分类存在

### 4.4 独有编码

- 视频编码含 VP9(7) 和 H.261(1)，大多数站点不包含这些
- 音频编码含 PCM(9) 和 Opus(6)，较罕见
- 编码名称简短：AVC（非 H.264/AVC）、HEVC（非 H.265/HEVC）

### 4.5 独有分辨率

- 4320p(7)：8K 超高清
- 1440p(5)：2K
- 540p(2)：低清

### 4.6 标签值跳跃

标签值在 5（驻站）和 7（中字）之间跳过值 6，在 16（特效）和 18（原声）之间跳过值 17。表明曾有标签被移除或保留。

### 4.7 制作组极简

仅3个选项（NONE/CBGWT/CBGER），是已采集站点中最少的之一。转载资源应选 NONE。

### 4.8 剧集完结要求

论坛规则明确要求剧集转载必须选择"完结"标签(3)，严禁分集发布（官组除外）。适配器应在转发剧集时自动添加完结标签。

## 六、适配器实现要点

### 6.1 倒序值映射

适配器必须使用藏宝阁特有的倒序值映射表，不能复用其他站点的映射：

```go
// 倒序映射示例：媒介
// UHD=9, Remux=8, Blu-ray=7, WEB-DL=6, ...
// 与大多数站点相反（通常 Blu-ray=7 向上递增）
```

### 6.2 字段名

```go
TorrentFileField: "file",                    // 标准

SubtitleField:   "small_descr",              // 标准
PtGenField:      "pt_gen",                   // 标准
DescrField:      "descr",                    // 标准
MediaInfoField:  "technical_info",           // 标准
AnonField:       "uplver",                   // 标准
TypeField:       "type",                     // 标准（无后缀）
MediumField:     "medium_sel[4]",            // 带 [4] 后缀

AudioCodecField: "audiocodec_sel[4]",        // 带 [4] 后缀
```

### 6.3 剧集自动标签

转发剧集时自动添加完结标签(3)：

```go
if isSeries(req.Type) {

```

### 6.4 分类映射需细化

因藏宝阁区分"动画"与"动漫"、"剧集"与"短剧"，适配器需要更细粒度的源分类映射逻辑。

## 七、审核脚本分析（CBGTorrentAuditAssistant v1.0.96）

### 7.1 脚本信息

| 项目 | 内容 |
|------|------|
| 名称 | 藏宝阁种子审核助手 (CBG Torrent Audit Assistant) |
| 版本 | 1.0.96 |
| 运行页面 | `details.php*`（详情页）+ `edit.php*`（编辑页）+ 审核弹窗页 |
| 权限 | GM_setValue / GM_getValue / GM_registerMenuCommand / GM_xmlhttpRequest |
| 审核API | `GET /web/torrent-approval-page` 获取 CSRF `_token`，`POST /web/torrent-approval` 提交 |

### 7.2 音频编码映射细节

脚本中 `AUDIO_MAPPING` 将所有 DTS 变体合并为 `DTS`(3)：

```
'dts-hd ma': 3, 'dts-hd': 3, 'dts:x': 3, 'dts': 3  → 全部 → DTS(3)
```

这意味着藏宝阁**不区分 DTS-HD MA / DTS-HD / DTS:X / DTS**，统一归为 DTS(3)。音频编码仅有 9 个选项（无 DTS-HD MA、无 DDP/E-AC3 以外的杜比变体）。

DDP 映射：`'e-ac3': 5, 'eac3': 5, 'ddp5.1': 5, 'ddp': 5, 'dd+': 5` → DDP(5)

### 7.3 不信任制作组（21个）

```
FGT, Hao4K, Mp4Ba, RARBG, GPTHD, SeeWeb, DreamHD, BlackTV, Xiaomi,
Huawei, MomoHD, DDHDTV, NukeHD, TagWeb, SonyHD, MiniHD, BitsTV,
ALT, XunLei, ZeroTV, LelveTV
```

标题含不信任制作组名称 → 错误。

### 7.4 官组检测

标题含 `CBGWT` 或 `CBGER`（不区分大小写，需独立词边界）→ 官方种子。

### 7.6 分集/合集/完结检测逻辑

**分集模式**（标题）：
- `S01E01`、`S01E01-E05`、`S01E01E02`、`EP01`、`EP01-05`、`第1集`、`E13-E21`、`E13,E21`

**合集模式**（标题）：
- `S01-S02`、`S01-02`、`第1-2季/集`、`合集`

**完结模式**（标题）：
- `全34集`、`全集/全季/已完结`、单独 `S01`（无E后续）、`Season 1`（无 episode）、`第1季`（无第X集）

**判定优先级**：如果检测到完结标识，则不算分集。如果检测到分集或合集，则不算完结。非官组的分集发布直接报错。

### 7.7 审核检查项

**错误（必须修复）**：

| # | 规则 |
|---|------|
| 1 | 副标题为空（所有分类必检） |
| 2 | 多集内容选了动画(404)分类 → 应选动漫(405) |
| 3 | 无多集信号选了动漫(405) → 应选动画(404) |
| 4 | 多集内容选了电影(401) → 应选剧集或其他 |
| 5 | 非官组检测到单集发布 → 禁止分集 |
| 6 | 未选择分类/媒介/编码/音频/分辨率（视频类） |
| 7 | 标题推断值与用户选择不一致（媒介/编码/音频/分辨率） |
| 8 | 视频类 MediaInfo 为空 |
| 9 | 官方种子未选制作组 |
| 10 | 标题含不信任制作组 |
| 11 | 非官方种子不可选官方标签 |
| 12 | 官方种子未选官方标签 |
| 13 | 简介未包含图片 |
| 14 | 简介图片无法访问 |
