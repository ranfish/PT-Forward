## 一、发种规范（通用发种规范 v1.3）

> 数据来源: 飞书 Wiki「通用发种规范」（试行 1.3 版本）
> 站点使用**自动种审**，规则判定可能存在不完美情况。

### 1.1 绝对禁止的资源

违反将导致种子被删除，情节严重者（明知故犯及多次发布者）将获警告直至删除账号。

1. **他站禁止转载资源**（举报奖励 2w 魔力，违者第一次警告，第二次 ban 号）
2. **9kg 资源**（美国 NC-17、香港 III 级、日本 R18、韩国 18 级影片及露点资源）
3. 质量较差视频（CAM/TC/TS/SCR/R5/HalfCD/MiniSD/MNHD/RMVB/RM/flv，针对电影）
4. 重复资源（同一来源完全相同，仅保留较早发布，断种例外）
5. 国内正在上映的电影资源
6. 其他站点规则中不允许的资源

### 1.2 限制发布的资源

违反将导致种子被删除：

1. **BT 转载资源**（仅允许种审组审核通过的）
2. **已完结电视剧必须以合集形式发布**，限制发布单集（同时适用于官组和用户）
3. **大包/合集申请制度**：体积超过 **1T** 或影视资源超过一部以上的合集，实行**申请发布制度**，需在论坛申请帖申请。**不允许体现个人兴趣的合集**（如"周星驰电影10部"）

### 1.3 主标题规范（0DAY 格式）

**标题严禁出现非英文字符以外的内容**（电子书分类豁免）。

**电视剧**：
```
英文名 S(季) 年份 分辨率 来源平台 来源方式 编码 画质技术 音频编码 声道数-制作组
```
范例：`The Glory S01 2023 2160p NF WEB-DL HEVC DV DDP 5.1-AGSVWEB`

各字段说明：

| 字段 | 说明 |
|------|------|
| 英文名 | 强制使用英文名，无英文名则使用拼音（首字母大写） |
| S(季) | 未完结剧集需标注 E(集) |
| 年份 | 多季统一使用首播季年份 |
| 分辨率 | 一季内包含多个分辨率需在简介说明 |
| 来源平台 | 国内平台不要填写 |
| 来源方式 | WEB-DL/Blu-ray/Remux 等 |
| 画质技术 | DV/HDR/SDR，仅 4K 及以上需要，1080p 无需填写 |
| 编码 | H.264/AVC 统一写 AVC，HEVC/H.265 统一写 HEVC，压制版写 x264/x265 |
| 音频编码 | DDP/AAC 等 |
| 声道数 | 2.0/5.1/7.1 等 |

**必须包含**：英文电影名称、上映年份、视频来源方式、音频编码方式、视频编码格式、制作组。电影不用包含"季"，来源平台和画质技术可不必包含。

**电影**：`英文名称 [年份] [剪辑版本] [发布说明] 分辨率 来源 [音频/]视频编码-发布组`
范例：`The Dark Knight 2008 PROPER 720p BluRay x264-SiNNERS`

**音乐专辑**：`艺术家本国名 - 专辑本国名 发行年份 - 文件格式 比特率 采样率 - 制作组(如有)`
范例：`林俊杰 - 伟大的渺小 2017 - FLAC 24bit 96khz`

**游戏**：`游戏英文名 [年份] [版本] [发布说明]-发布组`
范例：`Command And Conquer Red Alert 3 Uprising-RELOADED`

> **游戏类资源**：为防止病毒，仅限官方游戏发布/转载组发布。其他用户需至论坛人才招募区申请加入官组。

### 1.4 副标题规范

- 副标题为主标题补充信息，**不能为空**（必填）
- 副标题可以包含中文

### 1.5 资源简介要求

1. 电影资源**必须包含 MediaInfo**，且 MediaInfo 不必在正文中出现；MediaInfo 请使用**英文**，不应包含 BBCode
2. 电影电视剧**必须包含豆瓣链接、IMDb 链接**
3. **必须包含主海报一张和电影内容截图至少一张**（图片需外链有效，注意部分站点转载资源存在防盗链）
4. **必须使用 PT-Gen 获取简介**（`https://api.iyuu.cn/ptgen/`）
5. 合集类资源请包含简介和**完整资源目录截图**，MediaInfo 随机选取一个分集复制即可
6. 保留原发布者申明信息，但不得包含广告类信息，也无需包含个人信息和联系方式
7. 其他类型资源也请尽量包含文字类简介以及内容截图（至少一张）
8. 简介内禁止包含 MediaInfo 内容

### 1.6 资源类型选择

- 影视类资源**必须完整选择**：类型、媒介、编码、音频编码、分辨率、制作组
- 其他类型资源请勿选择影视类资源的信息
- 标签尽量包含

### 1.7 大包规则

- 超过 **1T** 的种子**必须标注"大包"标签**
- 不鼓励多发合集，特别是自行收集整理的合集（不利于辅种）

### 1.8 标签使用规则

- **非官组成员禁止使用官方标签**，违者警告
- **完结标签**：请审慎使用，如标题中出现分集，请勿使用完结标签
- 标签尽量包含

### 1.9 其他注意事项

1. 转载资源请勿更改资源内容、夹带私货，影响辅种——一经举报查实警告，多次 ban 号
2. 转载资源里请勿夹杂广告信息，如有发现将导致删种
3. 转载未完结的单集资源请尽量追更，保障剧集完整性；已完结剧集不鼓励转载分集
4. 被拒绝的种子请于 **7 日内**修正，长时间未修正的种子种审组保留删除权
5. 请勿改变或去除种审员对种子信息作出的修改（上传者可以修正错误）
6. 如果资源的原始发布信息基本符合规范，请尽量使用原始发布信息

### 1.10 通用资源发布罚则

#### 短时间大量不合规发布

| 次数 | 处罚 |
|------|------|
| 第 1 次 | 禁止上传权限 **3 天** |
| 第 2 次 | 警告 + 禁止上传权限 **30 天** |
| 第 3 次 | 警告 + 禁止上传权限 **300 天** |
| 第 4 次 | 禁止上传权限**终身** |

#### 非影视/音乐/游戏资源违规

非影视、音乐以及游戏资源的种子发布要求当前为 **Elite User 等级**，违反将触发删种加警告。

#### 拒绝种子不修改

对已发布被拒绝的资源请在 **7 天内**根据要求修改：

| 情节 | 处罚 |
|------|------|
| 轻微，PM 警告后及时配合改正 | 禁止上传权限 **7 天** |
| 严重，仍无视 PM | 禁止上传权限 **30 天** |
| 从不理睬 PM 且仍发布种子 | 禁止上传权限 **300 天** |

## 四、关键适配器设计要点

### 4.1 禁转资源检查

必须检查源站种子是否标记为"禁转"，AGSV 对禁转违规处罚严厉（第一次警告，第二次 ban 号）。

### 4.2 已完结电视剧必须合集

已完结电视剧禁止单集发布，适配器需检测剧集完结状态。

### 4.3 大包标签强制

种子体积 > 1T 必须选择"大包"标签（值=22）。

### 4.4 标题 0DAY 规范

标题必须使用英文（0DAY 格式），编码统一写法：AVC（非 H.264）、HEVC（非 H.265），压制版写 x264/x265。

### 4.5 MediaInfo 必填（影视类）

电影资源必须在 `technical_info` 字段填写 MediaInfo，且不应出现在简介正文中。

### 4.6 source_sel 和 processing_sel 缺失

无来源/地区/处理方式字段，质量字段仅 5 个（medium/codec/audiocodec/standard/team）。

### 4.7 官方标签限制

非官组成员（非 AGSVPT/AGSVWEB/AGSVMUS）禁止使用官方标签。

### 标题解析算法

```
1. 获取 h1#top 文本
2. 清除后缀标签：(已审)/(冻结)/(待定)/(免费)/(2X免费)/(推荐)/(热门)/(经典)/(禁止)/[2X]等
3. 清除「剩余时间...」和「(禁止)」
4. 检测「禁转」→ exclusive=1
5. 标题转小写 → title_lowercase
6. 正则匹配链：
   ├── 媒介(type)：
   │   WEB-DL→10, Remux→3, bluray+无编码词→Encode(7),
   │   webrip/dvdrip/bdrip→7, HDTV→5, UHD Blu-ray→11,
   │   bluray+有编码词→Blu-ray(1)
   ├── 编码(encode)：264/avc→1, 265/hevc→6, vc/vc-1→2, mpeg2→4, av1→12
   ├── 音频(audio)：flac→1, lpcm→10, ddp/eac3→19, aac→6, ac3/dd→11,
   │   truehd+atmos→17, dts-hd ma/dts-hd→8, dts:x/dtsx→18, dts→3
   ├── 分辨率(resolution)：1080p/i→1, 720p/i→3, 480p/i→4, 4k/2160p/uhd→5, 8k/4320p→6
   ├── 完结：`complete` → title_is_complete
   ├── 分集：`S\d+E\d+` → title_is_episode
   └── 制作组检测：agsv/goddramas/agsvmus/-hares/-rl/beitai
```

### MediaInfo 解析

```
1. MediaInfo 栏为空 → isMediainfoEmpty=true
2. 从 MediaInfo 提取：
   ├── Audio Language → isAudioChinese (Chinese/Mandarin)
   ├── Text Language → isTextChinese (Chinese) / isTextEnglish (English)
```

## 六、盒子规则

> 数据来源: 飞书 Wiki「盒子规则 - 理性刷流」

### 6.1 限速规则

| 用户类型 | 单种限速 | 超速后果 |
|----------|----------|----------|
| 未报备盒子 | **50 MiB/s** | 自动禁用下载权限 |
| 已报备盒子 | **95 MiB/s** | 自动禁用账号 |

### 6.2 盒子流量限制

- 已报备盒子：种子发布后 **72 小时内**，最多可获得种子体积 **3 倍**的上传量
- 发种人同样需要限制速度

### 6.3 刷流禁止

- **禁止以刷流为目的的拆包**
- **禁止秒删刷流**，跳车特别离谱的用户将受严厉处罚

### 6.4 超速封禁阶梯处罚

#### 盒子用户（超速 95 MiB/s）— 2 次解封机会

| 次数 | 处罚 |
|------|------|
| 第 1 次 | 警告 4 周 |
| 第 2 次 | **流量全部清空** + 警告 4 周 + 需完成超速复活考核 |
| 第 3 次 | **永久封禁，不予解封** |

#### 非盒子用户（超速 50 MiB/s）— 3 次解封机会

| 次数 | 处罚 |
|------|------|
| 第 1 次 | 警告 1 周 |
| 第 2 次 | 警告 2 周 |
| 第 3 次 | **流量全部清空** + 警告 2 周 + 需完成超速复活考核 |
| 第 4 次 | **永久禁用，不予解封** |

### 6.5 盒子报备方式

- 位置：**控制面板 → 设定首页 → seedbox 登记**
- IPv4 和 IPv6 **需要分别登记**
- 请勿清除曾经使用过但已不再使用的盒子信息
- 请保留盒子购买凭证以备管理组查看

### 6.6 羊毛盒禁止

## 七、上传规则

> 数据来源: 飞书 Wiki「上传规则」

### 7.1 上传总则

1. 上传者必须对上传的文件拥有合法的**传播权**
2. 上传者必须保证**上传速度与做种时间**。如果在其他人完成前撤种或做种时间**不足 24 小时**，或者故意低速上传，将被警告甚至取消上传权限
3. 对于自己发布的种子，发布者将获得**双倍的上传量**
4. 如果有一些违规但却有价值的资源，请将详细情况告知管理组，可能破例允许发布

### 7.2 上传者资格

- **任何人都能发布资源**
- 有些用户需要先在**候选区**提交候选
- **游戏类资源**：仅上传员及以上等级用户，或管理组特别指定用户可自由上传；其他用户必须先在候选区提交候选

### 7.3 允许的资源

| 类型 | 说明 |
|------|------|
| HD 视频 | 蓝光原碟/HD DVD 原碟/Remux、HDTV 流媒体、高清重编码（≥720p）、高清 DV |
| SD 视频 | 来源于高清媒介的标清重编码（≥480p）、DVDR/DVDISO、DVDRip/CNDVDRip |
| 无损音轨 | FLAC/APE 等 + 相应 CUE 表单 |
| 多声道音轨 | ≥5.1 声道的电影/音乐音轨（DTS/DTSCD 镜像等）、评论音轨 |
| PC 游戏 | 必须为**原版光盘镜像** |
| 预告片 | **7 日内**发布的高清预告片 |
| 软件/文档 | 与高清相关的软件和文档 |

### 7.4 不允许的资源

| 禁止项 | 说明 |
|--------|------|
| 总体积 < **100MB** | （高清软件/文档、单曲专辑例外） |
| SD upscale 视频 | 标清视频 upscale 或部分 upscale |
| 质量差 SD 视频 | CAM/TC/TS/SCR/DVDSCR/R5/R5.Line/HalfCD 等 |
| RealVideo 编码 | RMVB/RM 封装、flv 文件 |
| 单独样片 | 样片请和正片一起上传 |
| 有损音频 | <5.1 声道的有损 MP3/WMA 等（2.0+ 国语/粤语音轨例外） |
| 无 CUE 多轨音频 | 无正确 CUE 表单 |
| 游戏相关 | 硬盘版/高压版、非官方镜像、第三方 mod、小游戏合集、单独破解/补丁 |
| 压缩文件 | RAR 等压缩文件 |
| 重复资源 | Dupe 判定见 §7.5 |
| 敏感内容 | 色情、敏感政治话题等 |
| 损坏文件 | 读取或回放过程中出错的文件 |
| 垃圾文件 | 病毒/木马/网站链接/广告文档/种子中的种子文件 |

### 7.5 Dupe（重复）判定规则 — 质量重于数量

**来源优先级**：Blu-ray/HD DVD > HDTV > DVD > TV

- 同一视频高优先级版本使低优先级版本判定为重复
- 同一视频高清版本使标清版本判定为重复
- **动漫特例**：HDTV 版本和 DVD 版本有**相同优先级**
- 同来源同分辨率的高清重编码：按发布组确定优先级（Scene & Internal 排名）
- **总会保留**一个当前最佳画质来源经重编码而成的 **DVD5 大小**（~4.38GB）的版本
- 基于无损截图对比，高质量版本使低质量版本视为重复
- **不同区域/不同配音或字幕的 Blu-ray/HD DVD 原盘不视为重复**
- **每个无损音轨只保留一个版本**（分轨 FLAC 优先级最高）

**断种豁免**：

| 豁免条件 | 说明 |
|----------|------|
| 连续断种 **45 日以上** | 可发布新版本，不受 Dupe 约束 |
| 已发布 **18 个月以上** | 可发布新版本，不受 Dupe 约束 |

> 新版本发布后，旧的重复版本将被保留直至断种。

### 7.6 资源打包规则（试行）

**允许打包**：

| 资源类型 | 打包条件 |
|----------|----------|
| 电影合集 | 按套装售卖的高清电影合集（如 The Ultimate Matrix Collection Blu-ray Box） |
| 电视剧/综艺/动漫 | 整季打包 |
| 纪录片 | 同一专题 |
| 预告片 | 7 日内的高清预告片 |
| MV | 同一艺术家的 MV（标清按 DVD 打包，不允许单曲单独发布；高清需分辨率相同） |
| 音乐 | 同一艺术家 5 张及以上专辑方可打包；两年内发售的专辑可单独发布 |
| 动漫 | 分卷发售的剧集/角色歌/广播剧等 |
| 发布组打包 | 发布组打包发布的资源 |

**打包约束**：

- 打包时须**剔除站内已有资源**（或全部包括进来）
- 视频资源须**来源媒介相同**（如蓝光原碟）、**分辨率相同**（如 720p）、**编码一致**（如 x264），预告片例外
- 电影合集发布组也必须统一
- 音频资源须**编码格式一致**（如全部为分轨 FLAC）
- 打包发布后视情况删除相应单独种子

> 如对打包不明确，请咨询管理组。管理组保留解释权和处理权。

### 7.7 种子信息填写

**必须包含**：种子文件、主标题、副标题、IMDb 链接（如无不填）、基本信息（类型/媒介/编码等）、MediaInfo、影片简介、至少一张内容截图、标签。

**MediaInfo**：
- 尽可能规范，**不要使用中文**
- 不要在 MediaInfo 栏中重复填写
- 不要将 MediaInfo 放置到正文内容中

**外部信息**：电影和电视剧必须包含外部信息链接（如 IMDb），如存在的话。

**NFO**：NFO 图请写入 NFO 文件，不要粘贴到简介里。

## 八、H&R（Hit and Run）规则

> 数据来源: 飞书 Wiki「H&R规则」

目前站点**少部分资源**启用了 HR 规则。

### 8.1 H&R 定义

H&R = Hit and Run，即下载完资源后在规定时间内没有完成最少做种时间的行为（"下完就跑"）。

### 8.2 H&R 考核标准

| 条件 | 要求 |
|------|------|
| 豁免 | 下载不超过种子总大小的 **10%** 时，不计入 H&R 考核 |
| 考核期 | **480 小时（20 天）**内 |
| 通过条件 | 做种满 **72 小时（3 天）** 或 分享率达到 **3** |

### 8.3 H&R 上限

- H&R 上限数量为 **10 个**
- **超过上限账号将被封禁**

### 8.4 H&R 消除

可通过**冰晶**来消除 H&R。

## 十、审种脚本 vs 表单字段交叉比对

> 对比 Agsv-Torrent-Assistant.js v1.4.7 的常量映射与 upload.php 实际表单值，找出差异。

### 10.1 分类差异

| 分类 | 脚本 cat_constant | upload.php 表单 | 差异 |
|------|-------------------|----------------|------|
| 401-408 | ✓ 全部匹配 | 综合区 ✓ | 无差异 |
| **409 Misc(其他)** | **有** | **综合区无、学习区无** | **脚本有此分类，表单不可选** |
| 411-413 | ✓ 匹配 | ✓ | 无差异 |
| 415-419 | ✓ 匹配 | ✓ | 无差异 |

**结论**：409(Misc) 存在于脚本常量和详情页解析中，但 upload.php 的综合区/学习区下拉框均无此选项。可能为历史遗留分类或仅限管理操作。

### 10.2 媒介/编码/分辨率 — 完全一致

脚本 type_constant(10)、encode_constant(6)、resolution_constant(6) 与 §2.3-2.6 表单值完全一致，无差异。

### 10.3 音频编码差异

| 音频编码 | 脚本 audio_constant | upload.php §2.5 |
|----------|--------------------|--------------------|
| FLAC(1)~DDP/E-AC3(19) | ✓ 全部 15 个 | ✓ 匹配 |

**结论**：脚本不检测 ALAC 音频编码。如果种子选择了 ALAC，脚本无法验证其一致性。适配器转发时选择 ALAC(20) 提交是安全的（表单支持）。
