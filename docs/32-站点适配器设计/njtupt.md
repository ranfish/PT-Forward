## 一、发布页面表单字段分析

### 1.1 基础信息字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| `file` | file | ✓ | 种子文件 |
| `name` | text | - | 标题 |
| `small_descr` | text | - | 副标题 |
| `url` | text | - | IMDb 链接（`data-pt-gen="url"`） |
| `nfo` | file | - | NFO 文件 |
| `descr` | textarea | ✓ | 简介（BBCode） |
| `technical_info` | textarea | - | **MediaInfo**（独立字段） |
| `uplver` | checkbox | - | 匿名发布（value="yes"） |

### 1.2 PT-Gen 集成

支持四来源：IMDb + Douban + Bangumi + PT-Gen（`data-pt-gen` 出现在 `url` 和 `pt_gen` 属性中）。

### 1.3 缺失字段

- **`audiocodec_sel`**：无音频编码下拉框
- **`team_sel`**：无制作组下拉框
- **`source_sel`**：无来源字段
- **`processing_sel`**：无处理方式字段

- 值编号不连续（415 资料、422 音乐、425 游戏、428 软件）

## 二、关键适配器设计要点

### 2.1 MediaInfo 独立字段

浦园PT有独立的 MediaInfo 字段（`technical_info`，textarea 类型），适配器在转发时可自动填充此字段。

### 2.2 演出独立分类

有 **演出**（406）分类，演唱会/音乐剧/舞台剧资源应使用此分类而非音乐。

### 2.3 资料分类

有 **资料**（415）分类，教育/学习资料使用此分类。反映教育网特色。

### 2.4 极简质量字段

媒介 7 种、编码 6 种、分辨率 5 种，字段简洁。无音频编码和制作组下拉框。

### 2.5 PT-Gen 四来源

支持 IMDb + Douban + Bangumi + PT-Gen 四种来源，适配器可充分利用自动填充。

### 2.6 双 Tracker 域名

Tracker 地址同时包含两个域名：`njtupt.top` 和 `pt.online.njtech.edu.cn`。

### 2.7 标准 NexusPHP 规则

规则为标准 NexusPHP 模板（允许/禁止/dupe/打包规则均无特殊限制），无额外限制。
