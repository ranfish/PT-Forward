## 四、上传规则

### 4.1 上传总则

- 上传者必须对上传的文件拥有合法的传播权
- 做种时间不足 **24小时** 或故意低速上传，将被警告甚至取消上传权限
- 发布者将获得 **双倍上传量**

### 4.2 上传者资格

- 任何人都能发布资源
- 有些用户需要先在候选区提交候选
- **游戏类资源**：只有上传员及以上等级，或管理组特别指定用户，可自由上传；其他用户须候选

### 4.3 允许的资源

- 写真、视频
- PC 游戏

### 4.4 不允许的资源

- 总体积小于 **10MB** 的资源
- 重复(dupe)资源
- 涉及禁忌或敏感内容（如敏感政治话题等）
- 损坏的文件、垃圾文件

### 4.5 Dupe 判定规则

**来源媒介优先级**：合辑 > 单本/单个视频
- 同一视频高清版本使标清版本判定为重复
- 高优先级发布组版本使低优先级版本判定为重复
- 基于无损截图对比，高质量版本使低质量版本视为重复
- 来自其他区域、包含不同内容的版本不视为重复

**Dupe 豁免**：
- 新版本没有旧版本已确认错误/画质问题，或新版本来源质量更好 → 允许发布
- 旧版本连续断种 **45日以上** 或已发布 **18个月以上** → 不受 Dupe 约束
- 新版本发布后，旧的重复版本保留至断种

### 4.6 资源打包规则（试行）

允许打包：
- 按套装售卖的合集
- 整季电视剧/综艺节目/动漫
- 同一系列影片/写真
- 同一摄影师/厂商的视频/写真
- 发布组打包资源

打包要求：相同类型媒介、文件命名格式一致、发布组统一

### 4.7 标题命名规则

- 视频：`[中文名] 名称 [发布说明] [分辨率]`
  - 范例：`蝙蝠侠:黑暗骑士 The Dark Knight 720p`
- 写真：`[中文名] 名称 S**E** [发布说明]`
  - 范例：`Body Style 01 | Donghua Wei`
- 游戏：`[中文名] 名称 [年份] [版本] [发布说明][-发布组名称]`
  - 范例：`红色警戒3:起义时刻 Command And Conquer Red Alert 3 Uprising-RELOADED`

## 八、关键特点总结

1. **极简表单**：upload.php 仅有标题/副标题/简介/类型/标签，无任何质量字段（来源/媒介/编码/分辨率/制作组）
2. **双区域**：种子区（国产/日韩/欧美/南亚/中东/其他）+ 伪娘专区
3. **类型值异常**：`404=国产`（其他站通常404=纪录片）、`402=日韩`（其他站通常402=剧集）、`405=欧美`（其他站通常405=动漫），按地区而非内容类型分类
4. **50%随机Free**：随机促销中免费概率极高（50%）
5. **1个月永久2x**：所有种子发布1个月后自动永久2x上传
6. **无质量字段**：没有 source_sel/medium_sel/codec_sel 等字段，发种门槛极低
7. **Dupe 豁免**：断种45天或发布18个月以上
8. **认领宽松**：50人/种、500种/人、月300h达标
