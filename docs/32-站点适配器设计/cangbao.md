# 藏宝阁 (CBG / CangBao) 站点适配器设计

## 站点信息

| 项目 | 内容 |
|------|------|
| 站点名称 | 藏宝阁（CangBao / CBG） |
| 站点框架 | NexusPHP |
| 特殊功能 | MetaFlow 自动填表、标准模式系统(data-mode=4)、倒序字段值 |
| 规则页面 | 论坛发种规则帖 |

## 一、发布页面表单字段分析

**提交地址**: `takeupload.php`（POST multipart/form-data）

### 1.1 模式系统

使用 `data-mode='4'` 属性控制字段显示，字段名带 `[4]` 后缀。标准综合区模式。

### 1.2 基础信息字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| `file` | file | ✓ | 种子文件 |
| `name` | text | ✓ | 标题 |
| `small_descr` | text | - | 副标题 |
| `url` | text | - | IMDb 链接 |
| `pt_gen` | text | - | PT-Gen 链接 |
| `nfo` | file | - | NFO 文件 |
| `descr` | textarea | ✓ | 简介（BBCode） |
| `technical_info` | textarea | - | MediaInfo |
| `uplver` | checkbox | - | 匿名发布（value="yes"） |

**PT-Gen 支持**: 支持4种来源——imdb / douban / bangumi / indienova

### 1.3 MetaFlow 字段（独有）

表单中存在 MetaFlow 自动填表功能：
- DOM ID: `mf_input`（输入框）+ `mf_btn`（按钮）
- 由 `js/metaflow.js` 驱动
- **非标准表单字段**，不参与 POST 提交，仅用于前端自动填充其他字段

### 1.4 类型字段（`type`，data-mode=4）— 10个

**独特分类**: 短剧(403)、动画(404)、动漫(405)、儿童(406)、书籍(410)——比大多数 NexusPHP 站点分类更细分。注意"动画"与"动漫"是两个独立分类。

## 二、发种规则（论坛规则帖）

### 2.1 总则

- 推荐 4K 及更高分辨率资源
- 优先选择带简体中文字幕的资源
- 低于 1080p 的电影资源慎重发布

### 2.2 剧集规则

- 剧集转载**必须选择完结标签**，严禁分集发布（官方制作组除外）
- 从第一季开始发种，如无法持续更新则不要轻易发布

### 2.3 命名与元数据

- 遵循标准命名规范
- 文件完整性要求
- 准确的元数据填写
- 首发与重发原则

### 2.4 做种义务

- 发布者有长期做种义务

## 四、藏宝阁特殊注意事项

### 4.1 所有质量字段值倒序排列

藏宝阁是已采集站点中**唯一一个所有质量字段值从大到小倒序排列**的站点：
- 媒介：UHD=9 → VCD=1
- 编码：AV1=9 → H.261=1
- 音频：PCM=9 → MP3=1
- 分辨率：4320p=7 → 480P=1

这种设计使得值越大代表质量越高，在排序时自然有序。适配器映射时不能假设值与常见站点相同。

### 4.2 MetaFlow 自动填表

前端集成了 MetaFlow 工具（`js/metaflow.js`），可自动从 PT-Gen 链接提取信息填充表单。这不是表单字段，不影响 POST 数据，但意味着其他站点的种子可能需要重新映射字段。

### 4.3 分类细分

藏宝阁的分类比大多数 NexusPHP 站点更细：
- "动画"(404) 与 "动漫"(405) 是独立分类
- "短剧"(403) 独立于 "剧集"(402)
- "儿童"(406) 和 "书籍"(410) 作为独立分类存在

### 4.4 独有编码

- 视频编码含 VP9(7) 和 H.261(1)，大多数站点不包含这些
- 音频编码含 PCM(9) 和 Opus(6)，较罕见
- 编码名称简短：AVC（非 H.264/AVC）、HEVC（非 H.265/HEVC）

### 4.5 独有分辨率

- 4320p(7)：8K 超高清
- 1440p(5)：2K
- 540p(2)：低清

### 4.6 标签值跳跃

标签值在 5（驻站）和 7（中字）之间跳过了值 6，表明曾有标签被移除或保留。

### 4.7 制作组极简

仅3个选项（NONE/CBGWT/CBGER），是已采集站点中最少的之一。转载资源应选 NONE。

### 4.8 剧集完结要求

论坛规则明确要求剧集转载必须选择"完结"标签(3)，严禁分集发布（官组除外）。适配器应在转发剧集时自动添加完结标签。

## 六、适配器实现要点

### 6.1 倒序值映射

适配器必须使用藏宝阁特有的倒序值映射表，不能复用其他站点的映射：

```go
// 倒序映射示例：媒介
// UHD=9, Remux=8, Blu-ray=7, WEB-DL=6, ...
// 与大多数站点相反（通常 Blu-ray=7 向上递增）
```

### 6.2 字段名

```go
TorrentFileField: "file",                    // 标准
TitleField:      "name",                     // 标准
SubtitleField:   "small_descr",              // 标准
PtGenField:      "pt_gen",                   // 标准
DescrField:      "descr",                    // 标准
MediaInfoField:  "technical_info",           // 标准
AnonField:       "uplver",                   // 标准
TypeField:       "type",                     // 标准（无后缀）
MediumField:     "medium_sel[4]",            // 带 [4] 后缀
CodecField:      "codec_sel[4]",             // 带 [4] 后缀
AudioCodecField: "audiocodec_sel[4]",        // 带 [4] 后缀
ResolutionField: "standard_sel[4]",          // 带 [4] 后缀
TeamField:       "team_sel[4]",              // 带 [4] 后缀
TagsField:       "tags[4][]",                // 带 [4] 后缀，数组
```

### 6.3 剧集自动标签

转发剧集时自动添加完结标签(3)：

```go
if isSeries(req.Type) {
    req.Tags = append(req.Tags, "3") // 完结
}
```

### 6.4 分类映射需细化

因藏宝阁区分"动画"与"动漫"、"剧集"与"短剧"，适配器需要更细粒度的源分类映射逻辑。
