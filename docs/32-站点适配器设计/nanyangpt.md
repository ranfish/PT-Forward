## 〇、站点规则

> 数据来源: `rules.php` + `forums.php?topicid=20`（分类发布细则）采集 (2026-04-23)

### 0.1 账号保留规则

| 等级 | 保留条件 |
|------|---------|
| 大学讲师及以上 | 永远保留 |
| 优秀硕士+ | 封存账号后不会被删除 |
| 封存账号 | 连续 400 天不登录删除 |
| 未封存账号 | 连续 150 天不登录删除 |
| 无流量账号 | 连续 100 天不登录删除 |

### 0.2 账号封禁规则

- 注册多账号 → 禁止
- 作弊/严重违规 → 直接封禁
- 帮助被邀请人作弊 → 双方封禁
- 任何形式交易 → 永久封禁
- 盒子超速 **100 MiB/s** → 系统自动封禁
- 恶意刷流（大量下载+H&R 很少做种）→ 直接封禁

### 0.3 种子促销规则

**自动免费**：
- 文件总体积 > **50GB** → 自动免费
- 电视剧每季**第一集** → 自动免费
- **自压优质资源永久免费**

**其他促销**：
- 发种人上传一律 **2 倍上传**（永久）
- 关注度高的种子由管理员设置促销

### 0.4 上传总则

1. 发布者须在做种累计 **24 小时**且有至少 **3 个做种者**后方可撤种；72 小时后无人下载可撤种
2. 发布者获得**双倍上传量**
3. 违规种子**不经提醒直接删除**

> **重要**：未经允许，**禁止搬运发布蓝光原盘（包括 DIY）及 REMUX 类资源**。需在论坛申请报备通过后方可发布。

### 0.5 WEB-DL 限制

- 封装音轨仅允许：原始音轨 / 原始+英配 / 原始+国配(粤配/台配) / 原始+英配+国配
- 当有 **1080p** 版本存在时，**不接受 720p**
- 欧美电影/剧集仅接受国外流媒体源（Amazon、Netflix、iTunes 等）
- 不接受 WEB-DL 源的 DIY
- 原则上不接受 WEB-DL 源的压制作品
- 电影类：Blu-ray 重编码放出后，之前 WEB-DL 保留但不再接受新 WEB-DL

### 0.6 Dupe 规则

| 分类 | Dupe 策略 |
|------|----------|
| **电影类** | **不执行 dupe**（支持多家重编码） |
| **纪录片类** | **不执行 dupe** |
| **剧集类** | **执行 dupe**：同规格同来源只保留一版 |
| **动漫类** | 有详细 dupe 规则（见下） |

剧集 dupe 优先级：**高码 > 低码** / **软字幕 > 硬字幕** / **含国配版本优先**

### 0.7 码率最低要求

- 720p：不低于 **4000 kbps**
- 1080p：不低于 **8000 kbps**
- 720p 以下原则上不允许发布（DVD 原盘/DVDRip 除外）

### 0.8 合集规定

- 禁止搬运不规范合集包（私人打包、未完结作品打包等）
- 已完结剧集**不允许发布单集**，必须发布合集
- 合集中片源/格式/分辨率/制作组必须一致
- 原则上**禁止跨季/多季合集**
- 一天内发布单集 ≥ 3 集应打包发布

### 0.9 转载规则

- 转载须参照原发布者权利声明，要求注明出处的须注明
- 未允禁转的资源须先征得搬运许可
- **转载时一律禁止修改原文件名**

### 0.10 盒子规则

**禁用列表**（使用以下盒子账号将被封禁）：
- 共享 IP 的盒子
- Amazon / Azure / DigitalOcean / Google Cloud / Linode / Oracle / Vultr 云服务器
- 免费账号/试用账号/厂商赠送额度搭建的盒子

超速阈值：**100 MiB/s**

### 0.11 认领规则

- 每位会员最多认领 **30 个**种子，每个种子最多 **10 人**认领
- 月做种 **300 小时**（约 12.5 天）或月上传量达种子大小 **3 倍**
- 种子 > 1GB + 发布 > 1 个月 + 达标 → 获魔力值奖励
- 南洋出品奖励为常规资源的 **1.5 倍**

### 分类发布细则（forums.php?topicid=20）

#### 电影类

**允许**：HDTV、HDRip、BDRip、DVDISO、DVDRip 或同等画质正规制作组视频

**不允许**：韩语硬字幕、rmvb/rm/flv/3gp、dupe、upscaled、HR-HDTV、CAM/TS/TC/SCR/R5/HC、无意义合集

**命名规则**：
```
主标题：英文名.上映年代.介质.分辨率.视频编码-制作组
副标题：中文名 其它
例：This.Love.of.Mine.1986.BluRay.720p.HEVC.FLAC-NYHD
```

**简介必填**：海报 + 电影简介 + 编码信息(NFO/MediaInfo) + 电影截图

#### 剧集类

**命名规则**：
```
主标题：英文名.上映年代.介质.分辨率.视频编码-制作组
副标题：中文名 季度信息
例：Nirvana.in.Fire.Complete.1080p.WEB-DL.H264.AAC-NYHD
```

- 已完结剧集**必须发布合集**，不允许单集
- 全季合集仅在剧集确认完结时允许发布

#### 动漫类

有独立详细规则（置顶帖），覆盖动画/漫画/音乐/周边四类：

**命名格式**：`[动漫类别][中文名][英文名][集数][片源][字幕类型][分辨率][格式][字幕组]`

**Dupe 规则**：
- 同一集同一字幕组相同格式分辨率的简繁视为重复（简体优先）
- 片源不同不视为重复
- 同一专辑只允许一个无损+一个有损版本

**禁止事项**：低质量动漫(FLV/F4V/枪版/二压/KORSUB)、禁止发布的字幕组（猪猪字幕组、红旅字幕组、哈曼字幕组）、色情/18X、夹杂广告

#### 综艺/体育/纪录/音乐/学习/软件

各有独立命名规则和简介要求，详见 forums.php?topicid=20 原帖。

## 一、发布页面表单字段分析

### 1.1 基础信息字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| `cite_site` | text | - | **引用**：输入站内资源链接，一键引用（`id="cite_site"`） |
| `type` | select | ✓ | 类型/分类 |
| `file` | file | ✓ | 种子文件（id="torrent"） |
| `name` | text | - | 标题（规范填写，高级用户可见） |
| `small_descr` | text | - | 副标题 |
| `url` | text | - | IMDb 链接 |
| `dburl` | text | 建议 | **豆瓣链接**（字段名 `dburl`） |
| `nfo` | file | - | NFO 文件 |
| `descr` | textarea | ✓ | 简介（BBCode） |

### 1.2 缺失字段

以下常见字段在南洋PT **全部不存在**：

- **`medium_sel`**：无媒介下拉框
- **`codec_sel`**：无编码下拉框
- **`standard_sel`**：无分辨率下拉框
- **`audiocodec_sel`**：无音频编码下拉框
- **`team_sel`**：无制作组下拉框
- **`source_sel`**：无来源字段
- **`processing_sel`**：无处理方式字段
- **`uplver`**：无匿名发布选项
- **`pt_gen`**：无 PT-Gen 字段
- **标签系统**：无标签（tags）功能

**南洋PT是至今分析的所有站点中，发布页面质量字段最极简的站点之一**（与 Hamster/HUDBT 类似，无质量下拉框）。所有质量信息仅通过分类和标题传达。

### 1.3 类型（`type`）— 11 个分类

| 值 | 显示名称 |
|----|----------|
| 401 | 电影 |
| 402 | 剧集 |
| 403 | 动漫 |
| 404 | 综艺 |
| 405 | 体育 |
| 406 | 纪录 |
| 407 | 音乐 |
| 408 | 学习 |

| 410 | 游戏 |

**特点**：
- 11 个分类，标准编号 401-411
- 有 **学习**（408）分类
- 剧集使用 402，动漫使用 403
- 使用 "纪录" 而非 "纪录片"（406）

### 1.4 引用功能（cite_site）

南洋PT有独特的 **站内引用** 功能：
- 字段 `cite_site`：输入待引用的资源链接
- 按钮 `cite_button`："一键引用"
- 目前仅支持站内引用

此功能允许从已有种子直接复制信息，简化发布流程。

## 二、关键适配器设计要点

### 2.1 无质量字段

南洋PT发布页面 **没有任何质量下拉框**（媒介/编码/分辨率/音频/制作组/标签全部缺失）。适配器只需填写：
- `type`（分类）
- `file`（种子文件）
- `name`（标题——需规范填写）
- `url`（IMDb 链接）
- `dburl`（豆瓣链接）
- `descr`（简介）

这使得南洋PT成为 **最容易适配的目标站** 之一。

### 2.2 禁止搬运蓝光原盘/REMUX

规则明确：**未经允许，禁止搬运发布蓝光原盘（包括DIY）及REMUX类资源**。需在论坛申请报备。适配器在转发此类资源时应跳过南洋PT，或标记为需人工审核。

### 2.3 Dupe 规则（关键）

- **电影类、纪录片类**：**不执行 dupe**——支持多家重编码
- **剧集类**：**执行 dupe**——同规格同来源剧集包只保留一版
- 剧集 dupe 标准：1080p H264/265、2160p H265 等规格只留一版
- 优先级：高码 > 低码、软字幕 > 硬字幕、含国配优先

### 2.4 WEB-DL 限制

- 仅接受国外流媒体源（Amazon、Netflix、iTunes 等）
- 不接受 WEB-DL 源的 DIY
- 原则上不接受 WEB-DL 源的压制作品
- 音轨封装仅允许特定组合（原始/原始+英配/原始+国配等）
- 当 1080p 存在时，不接受 720p

### 2.5 码率最低要求

- 720p：不低于 4000kbps
- 1080p：不低于 8000kbps
- 720p 以下原则上不允许发布（DVD 除外）

### 2.6 豆瓣链接

南洋PT有独立的豆瓣链接字段（`dburl`），标记为"建议"填写。适配器可利用此字段。

### 2.7 候选制

有候选系统（offers.php），部分用户需先提交候选。

### 2.8 标题规范

标题需规范填写（链接到论坛 topicid=20 的规范帖），如：
```
银翼杀手 Blade Runner 1982 Final Cut 720p HDDVD DTS x264-ESiR
```

## 四、适配器转发策略建议

### 4.1 转发前过滤

适配器在向南洋PT转发前应执行以下检查：

1. **蓝光原盘/REMUX**：跳过或标记需人工审核
2. **WEB-DL DIY**：跳过
3. **WEB-DL 压制**：原则上跳过
4. **720p 以下**：跳过（DVD 除外）
5. **码率过低**：720p < 4000kbps 或 1080p < 8000kbps → 跳过

### 4.2 转发时填写

仅需填写 `type`、`file`、`name`、`url`、`dburl`、`descr`，无需处理任何质量映射。

### 4.3 剧集 Dupe 检查

转发剧集类资源前，应检索站内是否已有同规格同来源的版本。

## 五、全局转载策略对南洋PT的影响

| 规则 | 南洋PT 情况 | PT-Forward 处理 |
|------|-----------|-----------------|
| §30.5 禁止 9KG/成人 | 动漫区明确禁止 H 动漫/漫画；总则禁止色情 | 源站带成人标签→不转发 |
| §30.5 禁止禁转/独占 | 须征得原发布者搬运许可 | 源站带禁转标签→不转发 |
| §30.5 CatEDU 默认禁转 | 无特别规定 | 按全局规则处理 |
| 蓝光原盘/REMUX | 未经允许禁止搬运 | 此类资源不自动转发南洋 |
| 电影不 dupe | 支持多家重编码 | 可放心转发不同编码版本 |
| 剧集 dupe | 同规格只保留一版 | 转发前检索站内 |
| WEB-DL 限制 | 仅接受国外流媒体源 | 国内 WEB-DL 不转发南洋 |
| 码率要求 | 720p≥4Mbps, 1080p≥8Mbps | 低码率资源不转发 |
| 无质量下拉框 | 极简发布 | 只填分类+标题+简介+链接 |
| 豆瓣链接 | 有独立字段 | 从源站提取填入 dburl |
| Tracker | `https://tracker.nanyangpt.com/announce.php` | 制作种子时使用 |
