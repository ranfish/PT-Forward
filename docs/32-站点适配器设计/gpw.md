## 一、发布规则（rules.php?p=upload）

### 1.1 允许内容

| 编号 | 类型 | 说明 |
|------|------|------|
| 1.1.1 | 长片 | 时长 > 45 分钟的电影。介于长片和短片之间的查 IMDb |
| 1.1.2 | 短片 | 时长从数秒到约 45 分钟 |
| 1.1.3 | 现场表演 | 官方发行的音乐会/演艺/剧院/单口喜剧录像。**禁止盗录/直播抓流** |
| 1.1.4 | 电影集 | 仅限套盒/合辑中多部电影共用光盘且无法分离的原盘。Encode 及 Remux 必须分开发布 |

### 1.2 特别禁止

| 编号 | 禁止项 |
|------|--------|
| 1.2.1 | 预售类资源（CAM/TS/TC/R5/DVDScr） |
| 1.2.2 | **电视节目/电视连续剧** |
| 1.2.3 | 被 IMDb 添加成人标签的内容（如觉得不公正，发布前须获得批准） |
| 1.2.4 | MV 集锦 |
| 1.2.5 | 体育视频（体育纪录片允许） |
| 1.2.6 | 非官方影迷剪辑 |
| 1.2.7 | 视频教程/培训视频（电影制作相关须发布前获得批准） |
| 1.2.8 | 非视频种子、压缩格式（RAR/ZIP） |
| 1.2.9 | 打包电影（一种子一部电影，套盒仅原盘形式） |
| 1.2.10 | 电影与附加内容合在一起（必须分开发布，原盘除外） |
| 1.2.11 | **不允许劣质转码**（禁止二压，包括 BRRip 和二压 DVD5） |
| 1.2.12 | 低质量出品（见下方黑名单） |
| 1.2.13 | 预告片集锦 |
| 1.2.14 | **剧集**（电视连续剧/迷你剧/OVA 等） |
| 1.2.15 | 黑名单上的电影 |
| 1.2.16 | **禁止广告**（含封装时在视频音频标题处写入的推广链接等） |

## 二、格式规范

### 2.1 标清（SD）

| 编号 | 规则 |
|------|------|
| 3.1.1 | 标清指未达高清要求的种子 |
| 3.1.1.1 | 来自标清源的 x264 Encode 不允许放大 |
| 3.1.1.2 | 来自高清/超高清源：480p（max 854×480）或 576p（max 1024×576） |
| 3.1.2 | 必须使用 x264 + MKV |
| 3.1.4 | 禁止以 Encode 为源再压（二压） |
| 3.1.5 | 已有 720p 画面更好的蓝光 Encode 时，标清 Encode 将被删除（DVD 有实质性差异除外） |

### 2.2 高清（HD）

| 编号 | 规则 |
|------|------|
| 3.2.1 | 720p（max 1280×720）+ 1080p（max 1920×1080） |
| 3.2.2 | HD Encode：x264 + MKV（允许 HDR x265 1080p） |
| 3.2.4 | 必须源自 Blu-ray / HD-DVD / HDTV / WEB，其他源须获得批准 |

### 2.3 超高清（UHD）

| 编号 | 规则 |
|------|------|
| 3.3.1 | 2160p（max 4096×2160） |
| 3.3.2 | HDR/Dolby Vision 超高清源须保持此特性 + MKV |
| 3.3.3 | UHD Encode：x265 + MKV（Web 源 SDR 允许 x264） |

### 2.4 原盘

| 格式 | 容器 | 大小限制 |
|------|------|---------|
| DVD 原盘 | VOB_IFO（VIDEO_TS） | DVD5: 4.37 GiB / DVD9: 7.95 GiB |
| 蓝光原盘/DIY | M2TS 或 ISO | BD25: 23.28 / BD50: 46.57 / BD66: 61.47 / BD100: 93.13 GiB |
| HDTV 抓流 | TS 或 MKV | — |
| 蓝光 Remux | MKV | 须使用源光盘最优质量轨道 |

**Remux 混流顺序**（3.4.5.2）：视频 → 主音轨（标为默认）→ 次音轨 → 字幕

**Remux 音轨规则**（3.4.5.3）：
- 2.0 及以下 PCM → 须转 FLAC
- 24 bit DTS-HD MA 不可转 16 bit
- 2.1 及以上 PCM → 须转 DTS-HD MA / TrueHD / FLAC
- 允许 SRT 字幕
- 勾选「自制」须提供 eac3to log

### 2.5 附加内容（3.5）

| 编号 | 规则 |
|------|------|
| 3.5.1 | 附加内容：官方发行中的非电影主体视频材料（幕后花絮、采访等） |
| 3.5.1.1 | 必须勾选「非电影主体」选项 |
| 3.5.2 | 仅包含在官方零售发行的完整版时允许发布 |
| 3.5.3 | 仅含附加内容的光盘必须与主体光盘制成同一种子 |
| 3.5.4 | 拥有 IMDb 页面的附加内容必须单独发布 |

### 2.6 外挂字幕（3.6）

| 编号 | 规则 |
|------|------|
| 3.6.1 | 字幕应与对应种子视频文件同步，否则直接删除 |
| 3.6.2 | 允许格式：.sub .idx .sup .srt .vtt .ass .smi .ssa，压缩格式：.rar .zip .7z .tar .tgz .tar.gz |
| 3.6.3 | 建议文件名与视频文件一致，尾部可增加语言标记 |
| 3.6.4 | 字幕文件以 Unicode 编码为佳 |
| 3.6.5 | 单文件电影请直接上传单个字幕文件，不要打包多语种字幕 |

### 2.7 其他

| 编号 | 规则 |
|------|------|
| 4.1 | 不要发布没有完全访问权限的种子 |
| 4.2 | 两周内做种至少 48 小时或分享率达到 1 |
| 4.3 | 尽可能长期做种 |
| 4.4 | 做种时保证下载者能连上 |

## 三、槽位规则（rules.php?p=slots）

### 3.1 槽位总览

| 槽位类型 | Encode | Web | Remux | Untouched | DIY |
|----------|--------|-----|-------|-----------|-----|
| 槽位名称 | 中字质量/英字质量/特色/存档/Hi10p | SDR/DoVi | 质量/特色 | 原盘 | DIY |
| 编码 | x264/x265 | H.265 | — | — | — |

**按分辨率的完整槽位表**：

| 分辨率 | Encode x264 槽位 | Encode x265 槽位 | Web 槽位 | Remux 槽位 | 原盘槽 | DIY 槽 |
|--------|-----------------|-----------------|---------|-----------|-------|--------|
| SD | 中字+英字（共2） | — | — | — | 1 | — |
| 720p | 中字+英字+特色+存档+Hi10p（共5） | — | — | 1 | 1 | — |
| 1080p | 中字+英字+特色+存档+Hi10p（共5） | 中字+英字（共2） | — | 2（质量+特色） | 1 | 1 |
| 2160p | — | 中字+英字+特色+存档（共4） | SDR+DoVi（共2） | 2（质量+特色） | 1 | 1 |

### 3.2 槽位定义

| 槽位 | 字幕要求 | 适用 | 说明 |
|------|---------|------|------|
| **中字质量槽** | 必须内封中字（硬字/外挂不算） | 外语片 | 可入组合：中、中+英、中+原语言。其他组合进英字槽。含非国配冗余音轨不进此槽 |
| **英字质量槽** | 必须带英字（可封可挂，硬字不算） | 全部 | 非英语电影须内封或外挂英字。首个无字幕种子进此槽（非英语片标为「可替代」） |
| **特色槽** | 必须内封中字或带国配 | 仅外语片 | 以内容丰富度为主要考虑（特效字幕、国配音轨等）。须在描述中说明特色内容 |
| **存档槽** | 无要求 | 全部 | 以体积和压制质量为考虑，参照 0day/Scene。不允许冗余配音音轨 |
| **Hi10p 槽** | 无要求 | 动画电影 | 须发布前获得批准 |
| **Web SDR 槽** | 无要求 | SDR Web | 必须来自正规视频平台原始视频流，不得二压 |
| **Web DoVi 槽** | 无要求 | P5 杜比视界 | 仅 WEB-DL 有，不兼容 HDR10 |
| **Remux 质量槽** | 无要求 | 全部 | 首要考虑源音视频轨质量 |
| **Remux 特色槽** | 无要求 | 仅外语片 | **华语影片不允许特色 Remux** |
| **原盘槽** | 无要求 | 全部 | 必须是未经非必要改动的原始媒介数据复制品 |
| **DIY 槽** | 必须内封中字或带国配 | 仅外语片 | **华语影片不允许 DIY 原盘** |

**备注**：
- 中字：囊括了中文的多语字幕（如中英双语），一律按中字处理
- 国配：仅外语电影（不含粤语等方言），仅普通话（含台湾普通话）
- 方言：方言电影不能使用「国语配音」「特效字幕」标记
- 特效字幕：须提供至少两张特效截图（不计入基本三张截图要求，共计至少五张）

### 3.3 替代优先级

| 槽位 | 编码 | 考虑因素优先级 | 具体替代序列 |
|------|------|--------------|-------------|
| 中字质量槽 | x264 | 视频质量佳 > 体积小 | 须提交尽可能多的对比截图证明改进 |
| 中字质量槽 | x265 | 向下兼容 DoVi > 仅 HDR10 | WEB-DL 两个层应同源或同参源 |
| 英字质量槽 | x264 | 视频质量佳 > 体积小 | 同中字质量槽 |
| 英字质量槽 | x265 | 向下兼容 DoVi > 仅 HDR10 | 同中字质量槽 |
| 特色槽 | x264/x265 | 国配/特效字幕丰富 > 质量 > 视频质量优 | 内封中字/国配有其一 < 内封中字+国配 < 内封特效中字+国配 |
| 存档槽 | x264 | 视频质量佳 > 体积小 | 无内封中字 < 内封中字 |
| 存档槽 | x265 | 向下兼容 DoVi > 仅 HDR10 | 同上 |
| Remux 质量槽 | — | 原盘质量优 > 带内封中字 | 无中字普通源 < 内封中字普通源 < 无中字优质源 < 内封中字优质源。有无国配不纳入 |
| Remux 特色槽 | — | 国配/特效字幕丰富 > 视频质量优 | 无内封中字 < 内封中字 < 内封特效中字 < 内封中字+国配 < 内封特效中字+国配 |
| 原盘槽 | — | 原盘质量优 | 无中字普通源 < 内封中字普通源 < 无中字优质源 < 内封中字优质源。有无国配不纳入 |
| DIY 槽 | — | 国配/特效字幕丰富 > 视频质量优 | 同 Remux 特色槽 |

### 3.4 共存规则

**标清（2.1）**：1 个中字质量槽 + 1 个英字质量槽 = 共 2 个 x264 Encode 槽位

**高清（2.2）**：
- 5 个不同 x264 720p Encode + 5 个 x264 1080p Encode 可共存
- 存档槽 Encode 应至少比质量槽小约 20% 才能共存
- 另有 2 个 HDR x265 1080p 槽位（中字+英字），与上述独立
- 不允许 SDR x265 1080p（Hi10p 除外）

**超高清（2.3）**：
- 4 个 2160p Encode 可共存（中字+英字+特色+存档）
- 存档槽至少比质量槽小约 20%
- 提供足够对比截图证明优于高清源时，SDR 可占存档槽

**原盘（2.4）**：
- NTSC DVD + PAL DVD 各 1 个
- 720p：1 原盘 + 1 Remux
- 1080p/2160p：各 1 原盘 + 1 DIY + 2 Remux

**其他**：
- 每种剪辑版本拥有独立槽位组
- 外语电影国配种子进入特色槽
- 非英语电影英配种子可拥有额外英字质量槽
- 方言影片（不含国配/英配）可拥有额外中字质量槽

### 3.5 替代规则

**源替代顺序（3.1）**：
- SD：VHS < TV < HDTV | WEB < DVD < Blu-ray
- HD/UHD：HDTV < WEB | HD-DVD | Blu-ray
- 码率悬殊时蓝光/HD-DVD 可直接替代 TV/HDTV，无需对比图

**质量替代**：
- 首选编码器：x264（SD/HD）、x265（UHD）
- 来源信息未知的 H.264/H.265 可占 x264/x265 槽位但更易被替代
- 质量替代应尽可能多通过截图对比证明
- Scene 重大缺陷可被 REPACK/PROPER 自动替代

**不活跃（3.3）**：
- 不活跃超过 4 周自动可替代
- 发布 24 小时后仍未做种 → 标为「可替代」
- 尽量为不活跃种子续种而非替代

## 五、分享率规则（rules.php?p=ratio）

### 5.1 合格分享率计算

| 下载量 | 合格分享率（0% 做种） | 合格分享率（100% 做种） |
|--------|---------------------|----------------------|
| 0-5 GiB | 0.00 | 0.00 |
| 5-10 GiB | 0.15 | 0.00 |
| 10-20 GiB | 0.20 | 0.00 |
| 20-30 GiB | 0.30 | 0.05 |
| 30-40 GiB | 0.40 | 0.10 |
| 40-50 GiB | 0.50 | 0.20 |
| 50-60 GiB | 0.60 | 0.30 |
| 60-80 GiB | 0.60 | 0.40 |
| 80-100 GiB | 0.60 | 0.50 |
| 100+ GiB | 0.60 | 0.60 |

**公式**：合格分享率 = 最大合格分享率 × (1 - 做种数/完成数)

- **做种数**：过去一周做种时间 > 72 小时的平均做种数量
- **完成数**：已下载完成且未被系统删除的种子数量
- 下载量 < 5 GiB 不会被监控
- 下载量 ≥ 100 GiB 后合格分享率恒为 0.60

### 5.2 监控机制

- 下载 > 5 GiB 且分享率 < 合格分享率 → 两周改善期
- 监控期间又下载 10 GiB → 自动封禁下载权限
- 两周内未改善 → 失去下载权限
- 系统自动运行，管理员无法手动干预

## 六、积分规则（rules.php?p=bonus）

**积分公式**：积分/时 = 大小(GiB) × (0.025 + (0.06 × ln(1 + 做种时长) ÷ (做种数^0.6)))

**规律**：
- 保种体积越大、数量越多 → 积分越多
- 保种时间越长 → 积分越多
- 同种子做种人数越少 → 积分越多
- 唯一做种者 → 额外积分

积分可兑换：免费令牌、邀请、头衔等

## 十二、GPW 与 PT-Forward 关联分析

### 12.1 站点特色（与其他站点对比）

| 特征 | GPW | 常见 NexusPHP |
|------|-----|--------------|
| 框架 | **GazellePW**（SPA + MDX 规则） | NexusPHP |
| 内容范围 | **纯电影**（禁止剧集/体育/MV/教程） | 电影+剧集+综艺+动漫+... |
| 发布系统 | **槽位制**（精密的共存/替代体系） | 简单 Dupe 规则 |
| 发布前提 | **必须先查询 IMDB/豆瓣/TMDB 确认无重复** | 直接发布 |
| 质量字段 | 有下拉框（source/codec/resolution/processing） | medium/codec/resolution 下拉框 |
| 标签 | select 多选（35 个类型标签） | 复选框 |
| 字幕 | 外挂系统（独立上传，不放入种子） | 内嵌为主 |
| 版本信息 | 精细标签体系（3D/CC/MOC/HDR/DV 等） | 简单标签 |

### 12.2 适配器实现难点

1. **GazellePW 框架**：非标准 NexusPHP，API 完全不同
2. **槽位系统**：极度复杂的共存/替代判定逻辑（25 种可替代标记）
3. **发布前置检查**：须先通过 IMDB/豆瓣/TMDB 查询确认无重复
4. **纯电影**：仅接受电影类资源，剧集等禁止发布
5. **截图/描述要求极高**：至少 3 张原始分辨率 PNG + MediaInfo + 海报 + IMDb
6. **低质量制作组黑名单**：40+ 个制作组禁止发布
