## 一、发布规则（rules.php?p=upload）

### 1.1 允许内容

| 类型 | 说明 |
|------|------|
| 长片 | 时长 > 45 分钟的电影 |
| 短片 | 时长 < 45 分钟（数秒到约 45 分钟） |
| 现场表演 | 官方发行的音乐会/演艺/剧院/单口喜剧录像（**禁止盗录/直播抓流**） |
| 电影集 | 仅限套盒/合辑中多部电影共用光盘且无法分离的原盘 |

### 1.2 特别禁止

| 编号 | 禁止项 |
|------|--------|
| 1.2.1 | 预售类资源（CAM/TS/TC/R5/DVDScr） |
| 1.2.2 | **电视节目/电视连续剧** |
| 1.2.3 | 被 IMDb 添加成人标签的内容 |
| 1.2.4 | MV 集锦 |
| 1.2.5 | 体育视频（体育纪录片允许） |
| 1.2.6 | 非官方影迷剪辑 |
| 1.2.7 | 视频教程/培训视频 |
| 1.2.8 | 非视频种子、压缩格式（RAR/ZIP） |
| 1.2.9 | 打包电影（一种子一部电影，套盒仅原盘形式） |
| 1.2.10 | 电影与附加内容合在一起（必须分开发布，原盘除外） |
| 1.2.11 | **不允许劣质转码**（禁止二压，包括 BRRip 和二压 DVD5） |
| 1.2.13 | 预告片集锦 |
| 1.2.14 | **剧集**（电视连续剧/迷你剧/OVA 等） |
| 1.2.15 | 黑名单上的电影 |
| 1.2.16 | **禁止广告**（含封装时在视频音频标题处写入的推广链接等） |

### 1.3 低质量出品黑名单（禁止发布的制作组）

ALT, aXXo, BBATWEB, lackTV, BitsTV, BMDRu, BRrip, CM8, CrEwSaDe, CTFOH, CTRLHD, DDHDTV, DNL, DreamHD, ENTHD, **EVO**（WEB-DL 允许）, FaNGDiNG0, **FGT**, **FRDS**（Remux 允许）, HD2DVD, HDTime, HDT（Remux）, Huawei, GPTHD, ION10, iPlanet, KiNGDOM, Leffe, Mp4Ba, mHD, MiniHD, mSD, **MOMOWEB**, nHD, nikt0, NSBC, nSD, NhaNc3, NukeHD, OFT, PRODJi, **RARBG**, RDN, SANTi, SeeHD, SeeWEB, SM737, SonyHD, STUTTERSHIT, TAGWEB, ViSION, **VXT**, WAF, x0r, Xiaomi, **YIFY**, PSP/iPad/移动设备预设 Encode

### 1.4 命名规则

- 文件（夹）名必须使用电影**原始语种名称**或**官方英文名**
- Internal Remux 文件名须包含原始发行年、分辨率、音视频编码
- 无意义内容（序号、个人标记）→ 标为「可替代」
- 中字槽种子文件名中带非中/英字符 → 标为「可替代」
- 压制组发行不应重命名（除非不符合命名规则）
- DVD/BD 文件目录结构不允许改动，仅顶层文件夹允许重命名
- **不要在种子内包含字幕文件，应单独上传**

### 1.5 截图与描述要求

- **截图：至少 3 张与影片分辨率相同的 PNG 格式截图**
- **允许图床**：kshare.club, pixhost.to, ptpimg.me, img.pterclub.net, img.pterclub.com, yes.ilikeshots.club, imgbox.com, s3.pterclub.com
- **MediaInfo**：必须提供（蓝光原盘用 BDInfo）
- **电影海报**：必须提供封面图，优先官方艺术海报和影院海报
- **IMDb 链接**：必须填写（如存在）
- **版本信息**：非原始版本须勾选版本信息标签
- **字幕信息**：必填
- **标签**：英文逗号分隔，英文点号分隔单词（如 sci.fi, post.rock）

## 二、格式规范

### 2.1 标清（SD）

- x264 + MKV
- 来自标清源：不可放大
- 来自高清/超高清源：480p（max 854×480）或 576p（max 1024×576）
- 禁止以 Encode 为源再压（二压）

### 2.2 高清（HD）

- **720p**（max 1280×720）+ **1080p**（max 1920×1080）
- HD Encode：x264 + MKV（允许 HDR x265 1080p）
- 来源：Blu-ray / HD-DVD / HDTV / WEB

### 2.3 超高清（UHD）

- **2160p**（max 4096×2160）
- HDR/Dolby Vision 超高清源须保持此特性 + MKV
- UHD Encode：x265 + MKV（Web 源 SDR 允许 x264）

### 2.4 原盘

| 格式 | 容器 | 大小限制 |
|------|------|---------|
| DVD 原盘 | VOB_IFO（VIDEO_TS） | DVD5: 4.37 GiB / DVD9: 7.95 GiB |
| 蓝光原盘/DIY | M2TS 或 ISO | BD25: 23.28 / BD50: 46.57 / BD66: 61.47 / BD100: 93.13 GiB |
| HDTV 抓流 | TS 或 MKV | — |
| 蓝光 Remux | MKV | 须使用源光盘最优质量轨道 |

**Remux 混流顺序**：视频 → 主音轨（标为默认）→ 次音轨 → 字幕

**Remux 音轨规则**：
- 2.0 及以下 PCM → 须转 FLAC
- 24 bit DTS-HD MA 不可转 16 bit
- 2.1 及以上 PCM → 须转 DTS-HD MA / TrueHD / FLAC
- 允许 SRT 字幕
- 勾选「自制」须提供 eac3to log

## 三、槽位规则（rules.php?p=slots）

### 3.1 槽位总览

每部影视作品在无特殊版本时，完整槽位表如下：

| 槽位类型 | Encode | Web | Remux | Untouched | DIY |
|----------|--------|-----|-------|-----------|-----|
| 槽位名称 | 中字质量/英字质量/特色/存档/Hi10p | SDR/DoVi | 质量/特色 | 原盘 | DIY |
| 编码 | x264/x265 | H.265 | — | — | — |

**按分辨率的槽位数量**：

| 分辨率 | Encode 槽位数 | Remux | 原盘 | DIY |
|--------|-------------|-------|------|-----|
| SD | 2（x264 中字+英字） | — | 1 | — |
| 720p | 5（x264 五类） | 1 | 1 | — |
| 1080p | 7（x264 五类 + x265 中字/英字） | 2 | 1 | 1 |
| 2160p | 4（x265 四类） | 2 | 1 | 1 |

### 3.2 槽位定义

| 槽位 | 字幕要求 | 说明 |
|------|---------|------|
| **中字质量槽** | 必须内封中字（硬字/外挂不算） | 以压制质量为考虑因素 |
| **英字质量槽** | 必须带英字 | 非英语电影须内封或外挂英字 |
| **特色槽** | 必须内封中字或带国配 | 以内容丰富度为主要考虑（特效字幕/国配音轨等） |
| **存档槽** | 无要求 | 以体积和压制质量为考虑，参照 0day/Scene |
| **Hi10p 槽** | 无要求 | 专供动画电影，须获批 |
| **Web SDR 槽** | 无要求 | 专供 SDR Web 资源 |
| **Web DoVi 槽** | 无要求 | 专供 P5 杜比视界（仅 WEB-DL） |
| **Remux 质量槽** | 无要求 | 以源音视频轨质量为考虑 |
| **Remux 特色槽** | 无要求 | 以特色内容为考虑（**华语影片不允许**） |
| **原盘槽** | 无要求 | 以原盘音视频轨质量为考虑 |
| **DIY 槽** | 必须内封中字或带国配 | 以特色内容为考虑（**华语影片不允许**） |

### 3.3 替代优先级

| 槽位类型 | 考虑因素优先级 |
|----------|--------------|
| 质量槽（x264） | 视频质量佳 > 体积小 |
| 质量槽（x265） | 向下兼容 DoVi（HDR10 层同等或更优）> 仅 HDR10 |
| 特色槽 | 国配/特效字幕丰富 > 质量 > 视频质量优 |
| 存档槽 | 视频质量佳 > 体积小 |
| Remux 质量槽 | 原盘质量优 > 带内封中字 |
| Remux 特色槽 | 无中字 < 内封中字 < 内封特效中字 < 内封中字+国配 < 内封特效中字+国配 |
| 原盘槽 | 原盘质量优（有无国配不纳入） |

### 3.4 可替代标记（25 种）

问题纵横比、非原始纵横比、无谓高码（含音轨码率上限表）、音轨冗余、反交错问题、帧率错误、字幕不同步、格式不当、分辨率不当、劣质源、低质量、播放问题、残缺、缺少基本字幕、未强制英文字幕、无原声音轨、音轨不同步、问题裁边、劣质字幕翻译、硬字幕、劣质转码、含水印、放大、不活跃（自动）、冗余文件

## 五、分享率规则（rules.php?p=ratio）

### 5.1 合格分享率计算

| 下载量 | 合格分享率（0% 做种） | 合格分享率（100% 做种） |
|--------|---------------------|----------------------|
| 0-5 GiB | 0.00 | 0.00 |
| 5-10 GiB | 0.15 | 0.00 |
| 10-20 GiB | 0.20 | 0.00 |
| 20-30 GiB | 0.30 | 0.05 |
| 30-40 GiB | 0.40 | 0.10 |
| 40-50 GiB | 0.50 | 0.20 |
| 50-60 GiB | 0.60 | 0.30 |
| 60-80 GiB | 0.60 | 0.40 |
| 80-100 GiB | 0.60 | 0.50 |
| 100+ GiB | 0.60 | 0.60 |

**公式**：合格分享率 = 最大合格分享率 × (1 - 做种数/完成数)

- **做种数**：过去一周做种时间 > 72 小时的平均做种数量
- **完成数**：已下载完成且未被系统删除的种子数量
- 下载量 < 5 GiB 不会被监控
- 下载量 ≥ 100 GiB 后合格分享率恒为 0.60

### 5.2 监控机制

- 下载 > 5 GiB 且分享率 < 合格分享率 → 两周改善期
- 监控期间又下载 10 GiB → 自动封禁下载权限
- 两周内未改善 → 失去下载权限
- 系统自动运行，管理员无法手动干预

## 六、积分规则（rules.php?p=bonus）

**积分公式**：积分/时 = 大小(GiB) × (0.025 + (0.06 × ln(1 + 做种时长) ÷ (做种数^0.6)))

**规律**：
- 保种体积越大、数量越多 → 积分越多
- 保种时间越长 → 积分越多
- 同种子做种人数越少 → 积分越多
- 唯一做种者 → 额外积分

积分可兑换：免费令牌、邀请、头衔等

## 十一、GPW 与 PT-Forward 关联分析

### 11.1 站点特色（与其他站点对比）

| 特征 | GPW | 常见 NexusPHP |
|------|-----|--------------|
| 框架 | **GazellePW**（SPA + MDX 规则） | NexusPHP |
| 内容范围 | **纯电影**（禁止剧集/体育/MV/教程） | 电影+剧集+综艺+动漫+... |
| 发布系统 | **槽位制**（精密的共存/替代体系） | 简单 Dupe 规则 |
| 分类 | 无传统分类下拉框 | type 下拉框 |
| 质量字段 | 无下拉框（通过槽位系统管理） | medium/codec/resolution 下拉框 |
| 标签 | 自由文本（英文逗号分隔） | 复选框 |
| 字幕 | 外挂系统（独立上传，不放入种子） | 内嵌为主 |
| 版本信息 | 精细标签体系（3D/CC/MOC/HDR/DV 等） | 简单标签 |

### 11.2 适配器实现难点

1. **GazellePW 框架**：非标准 NexusPHP，API 完全不同
2. **槽位系统**：极度复杂的共存/替代判定逻辑（25 种可替代标记）
3. **无标准下拉框**：分类/质量信息通过不同机制收集
4. **纯电影**：仅接受电影类资源，剧集等禁止发布
5. **截图/描述要求极高**：至少 3 张原始分辨率 PNG + MediaInfo + 海报 + IMDb
6. **低质量制作组黑名单**：40+ 个制作组禁止发布
