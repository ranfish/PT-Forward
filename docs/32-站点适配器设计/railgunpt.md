# 轨道炮 站点适配器设计

## 站点信息

| 项目 | 内容 |
|------|------|
| 站点名称 | 轨道炮（RailgunPT） |
| 站点地址 | https://bilibili.download |
| 站点框架 | NexusPHP |
| Tracker URL | `https://bilibili.download/announce.php` |
| 特殊功能 | 下架视频备份分类(420)、漫画分类(419)、学习分类(411)、双模式(mode 4/5)、候选制、认领系统(1000种)、趣味盒 |
| 规则页面 | rules.php |
| PT-Gen 来源 | imdb / douban / bangumi / indienova |
| 官方 TG 群 | https://t.me/RailgunPT |

**站点角色**: 无官组，**只能做目标站（发布站），不能做源站**。

**用户指示 — 发布范围**：
- ✅ **只允许发布至种子区**（mode=4，14 个分类）
- ❌ **禁止发布至特别区**（mode=5，6 个成人分类）
- 站点导航栏显示"种子9kg"入口，进一步确认特别区为 9KG 内容

**⚠ 成人版块禁止规则（§30.3）**：
- 轨道炮特别区（mode=5）包含 6 个成人分类：国产(413)、日韩(414)、欧美(415)、H漫(416)、黄油(417)、写真(418)
- **禁止下载和发布这些分类的资源**，YAML `skip_categories` + `skip_modes: ["5"]`
- 启动时由 RequiredSkips 审计（§30.4.2）

## 一、发布页面表单字段分析

**提交地址**: `takeupload.php`（POST multipart/form-data）

### 1.1 模式系统

使用 `data-mode='4'`（种子区，✅ 允许发布）和 mode=5（⛔ 特别区/成人区，用户明确指示禁止发布，§30.3）。两个字段集共享相同的值（mode 5 无 audiocodec_sel 和 2K 分辨率）。

### 1.2 基础信息字段

| 字段名 | 字段类型 | 必填 | 说明 |

| `file` | file | ✓ | 种子文件 |
| `name` | text | - | 标题 |
| `small_descr` | text | - | 副标题 |

## 三、RailgunPT 特殊注意事项

### 3.1 仅目标站

无官组，只能做目标站。在 PT-Forward 中应标记为 `SourceEnabled=false`。

### 3.2 下架视频备份分类

独特的"下架视频备份"(420)分类，与站点 bilibili.download 域名定位一致。适配器可将来自己知下架平台的资源归入此分类。

### 3.3 无制作组字段

表单中无 `team_sel`，发布时无需选择制作组。

### 3.4 无 MediaInfo 字段

无 `technical_info` 独立字段，MediaInfo 应写在简介中。

### 3.5 音频编码合并

TrueHD + Atmos 合并为 TrueHD/Atmos(1)，DTS-HD + DTS-HD MA 合并为 DTS-HD/DTS-HDMA(2)。无独立 Atmos、DTS:X、DDP 选项。

### 3.6 双模式

mode 4（影视）有额外 `audiocodec_sel` 字段，mode 5（非影视）无音频编码字段。其他字段（媒介/编码/分辨率）值完全相同。

### 3.7 媒介值倒序

媒介值从 1（Blu-ray，最高质量）到 9（Track），非按数字大小排列质量等级。

```go
    if isDelisted {
        return 420 // 下架视频备份
    }
    switch category {
    case "Movies": return 401
    case "TV Series": return 402
    // ...
    }
}
```

### 5.3 音频编码合并匹配

```go
func mapAudioCodec(codec string) int {
    switch {
    case strings.Contains(codec, "Atmos") || strings.Contains(codec, "TrueHD"):
        return 1  // TrueHD/Atmos
    case strings.Contains(codec, "DTS-HD MA") || strings.Contains(codec, "DTS-HD"):
        return 2  // DTS-HD/DTS-HDMA
    case strings.Contains(codec, "DTS"):
        return 2   // DTS 也归入 DTS-HD/DTS-HDMA
    case strings.Contains(codec, "AC3") || strings.Contains(codec, "DD"):
        return 3
    case strings.Contains(codec, "FLAC"):
        return 5
    case strings.Contains(codec, "MP3"):
        return 6
    case strings.Contains(codec, "AAC"):
        return 7
    default:
        return 9  // Other
    }
}
```

## 六、发种规则（rules.php）

### 6.1 上传总则

- 上传者必须对上传的文件拥有合法的传播权
- 上传者必须保证上传速度与做种时间——如果在其他人完成前撤种或做种时间不足 24 小时，或者故意低速上传，将被警告甚至取消上传权限
- 发布者获得**双倍上传量**
- 违规但有价值的资源可联系管理组破例

### 6.2 上传者资格

- **任何人都能发布资源**
- 有些用户需先在候选区提交候选
- **游戏类资源**：仅上传员及以上等级或管理组特别指定用户可自由上传，其他用户必须先在候选区提交候选

### 6.3 允许的资源

- **高清（HD）视频**：完整高清媒介（蓝光/HD DVD 原碟或 remux）、HDTV 流媒体、来源于上述媒介的高清重编码（≥720p）、其他高清视频（如高清 DV）
- **标清（SD）视频**：仅限来源于高清媒介的标清重编码（≥480p）、DVDR/DVDISO、DVDRip/CNDVDRip
- **无损音轨**（及 cue 表单）：FLAC、Monkey's Audio 等
- **5.1 声道或以上**的电影音轨、音乐音轨（DTS、DTS CD 镜像等）、评论音轨
- **PC 游戏**（必须为原版光盘镜像）
- **7 日内高清预告片**
- 高清相关软件和文档

### 6.4 不允许的资源

- **总体积 < 100MB** 的资源
- 标清视频 upscale 或部分 upscale 而成的视频
- 标清级别质量较差的视频（CAM、TC、TS、SCR、DVDSCR、R5、R5.Line、HalfCD 等）
- **RealVideo 编码**的视频（RMVB/RM）、**flv 文件**
- 单独的样片（须和正片一起上传）
- **未达到 5.1 声道**的有损音频（MP3、WMA 等）
- 无正确 cue 表单的多轨音频
- 硬盘版/高压版游戏、非官方游戏镜像、第三方 mod、小游戏合集、单独破解或补丁
- **RAR 等压缩文件**
- 重复（dupe）资源
- 禁忌或敏感内容（色情、敏感政治话题等）
- 损坏的文件、垃圾文件（病毒/木马/广告/种中种等）

### 6.5 Dupe 判定规则

**优先级**：Blu-ray/HD DVD > HDTV > DVD > TV（高优先级使低优先级成为 dupe）
- 同一视频高清版本使标清版本成为 dupe
- **动漫特例**：HDTV 版本和 DVD 版本有**相同优先级**
- 相同媒介、相同分辨率的高清重编码：按发布组确定优先级（参考 "Scene & Internal, from Group to Quality-Degree"）
- **总会保留一个 DVD5 大小（≈4.38GB）的最佳画质重编码版本**
- 基于无损截图对比，高质量版本使低质量版本成为 dupe
- 不同区域/不同配音/字幕的 Blu-ray/HD DVD 原盘**不视为 dupe**
- 每个无损音轨原则上只保留一个版本，**分轨 FLAC 优先级最高**

**新版本规则**：
- 新版本无旧版已确认错误/画质问题，或来源质量更好 → 允许发布，旧版成 dupe
- 旧版连续断种 **45 日以上**或已发布 **18 个月以上** → 新版不受 dupe 约束
- 新版发布后，旧版保留至断种

### 6.6 资源打包规则（试行）

**允许打包**：
- 按套装售卖的高清电影合集（如 The Ultimate Matrix Collection Blu-ray Box）
- 整季电视剧/综艺/动漫
- 同一专题纪录片
- 7 日内高清预告片
- 同一艺术家 MV（标清 MV 只允许按 DVD 打包，单曲不允许单独发布；高清 MV 分辨率相同可打包）
- 同一艺术家音乐（5 张以上专辑方可打包；两年内发售的可单独发布；打包时须剔除站内已有或全部包括）
- 分卷发售的动漫剧集、角色歌、广播剧等
- 发布组打包发布的资源

**打包要求**：
- 视频须相同媒介、相同分辨率、相同编码（预告片例外），电影合集发布组也须统一
- 音频须相同编码（如全部为分轨 FLAC）
- 打包后视情况删除相应单独种子

### 6.7 种子信息规范

**标题格式**（0day 命名法）：
- 电影：`[中文名] 名称 [年份] [剪辑版本] [发布说明] 分辨率 来源 [音频/]视频编码-发布组名称`
  - 例：`蝙蝠侠:黑暗骑士 The Dark Knight 2008 PROPER 720p BluRay x264-SiNNERS`
- 电视剧：`[中文名] 名称 [年份] S**E** [发布说明] 分辨率 来源 [音频/]视频编码-发布组名称`
  - 例：`越狱 Prison Break S04E01 PROPER 720p HDTV x264-CTU`
- 音轨：`[中文艺术家名 - 中文专辑名] 艺术家名 - 专辑名 [年份] [版本] [发布说明] 音频编码[-发布组名称]`
  - 例：`恩雅 - 冬季降临 Enya - And Winter Came 2008 FLAC`
- 游戏：`[中文名] 名称 [年份] [版本] [发布说明][-发布组名称]`
  - 例：`红色警戒3:起义时刻 Command And Conquer Red Alert 3 Uprising-RELOADED`

**副标题**：不要包含广告或求种/续种请求

**外部信息**：电影和电视剧必须包含 IMDb 链接（如果存在）

**简介要求**：
- 电影/电视剧/动漫：必须包含海报/横幅/封面；尽量包含截图、详细格式信息（格式/时长/编码/码率/分辨率/语言/字幕）、演职员名单及剧情概要
- 体育节目：不得在文字介绍或截图/文件名/文件大小/时长中泄露比赛结果
- 音乐：必须包含专辑封面和曲目列表
- PC 游戏：必须包含海报或封面；尽量包含截图
- NFO 图请写入 NFO 文件，不要粘贴到简介

**例外条款**：
- 允许来源于 TV/DSR 的体育类标清视频
- 允许 < 100MB 的高清相关软件和文档
- 允许 < 100MB 的单曲专辑
- 允许 2.0 声道或以上的国语/粤语音轨
- 允许附带字幕、游戏破解与补丁、字体、包装扫描图（统一打包或统一不打包）
- 允许发布音轨时附带附赠 DVD 相关文件

## 七、促销规则

### 7.1 随机促销（发布时系统自动）

| 概率 | 促销类型 |
|------|---------|
| 10% | 50% 下载 |
| 5% | 免费 |
| 5% | 2x 上传 |
| 3% | 50% 下载 & 2x 上传 |
| 1% | 免费 & 2x 上传 |
| 76% | 无促销 |

### 7.2 自动促销规则

- 文件总体积 > 20GB → 自动"免费"
- Blu-ray Disk / HD DVD 原盘 → 自动"免费"
- 电视剧等每季第一集 → 自动"免费"
- 关注度高的种子由管理员设置促销

### 7.3 促销时限

- 除"2x 上传"无时限外，其余促销限时 **7 天**
- **所有种子发布 1 个月后自动永久成为"2x 上传"**
- 不定期开启全站免费
