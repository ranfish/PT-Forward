## 一、发布页面表单字段分析

**提交地址**: `takeupload.php`（POST multipart/form-data）

**字段名无后缀**（裸名，如 `medium_sel` 而非 `medium_sel[4]`）。

**发布流程**: 非官方组用户需先在候选区提交候选，审核通过后方可发布。

### 1.1 基础信息字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| `file` | file | ✓ | 种子文件 |
| `name` | text | - | 标题（0day 命名法，发布后自动去除点号） |
| `small_descr` | text | - | 副标题（中文名、字幕信息、音轨信息等） |
| `douban_id` | text | - | 豆瓣 ID/链接（可输入豆瓣影视 ID 或完整链接） |
| `descr` | textarea | ✓ | 简介（BBCode，需含豆瓣简介 + MediaInfo + 截图） |
| `uplver` | checkbox | - | 匿名发布（值=yes） |
| `hr` | checkbox | - | HR 标记（值=1） |
| `waitcheck` | checkbox | - | 等待审核（值=1） |
| `banned` | checkbox | - | 禁转标记（值=yes） |

注意：
- **无 PT-Gen 字段**，需手动在 `descr` 中填入豆瓣信息。
- **无 `url`（IMDb 链接）字段**，仅 `douban_id`。
- **无 `source_sel`、`processing_sel`、`team_sel` 字段**——这是 PTerClub 的显著特征。
- 原盘类资源必须包含 BDInfo，其他资源须包含 MediaInfo。
- 标题命名规范：`电影名.年份.分辨率.媒介.视频编码.音频编码-发布小组名`。

- 规则页明确：**"动画"资源类型按电影和剧集属性选择分类，动画仅通过标签体现**。

### 1.3 缺失字段

| 字段 | 说明 |
|------|------|
| `source_sel` | 无来源媒介字段 |
| `processing_sel` | 无处理方式/地区字段 |
| `team_sel` | 无制作组下拉（靠标题 0day 命名中的小组名） |
| `pt_gen` | 无 PT-Gen（手动在简介中填豆瓣信息） |
| `url`（IMDb） | 无 IMDb 链接字段（仅有豆瓣） |
| `technical_info` | 无独立 MediaInfo 字段（写在简介 `descr` 中） |

## 二、发布规则要点

### 2.1 候选制

- 非官方组用户需先在候选区提交候选。
- 候选区当前有 7 个待审项（导航栏显示"候选(7)"）。

### 2.2 标题命名（0day 命名法）

影视资源必须使用 0day 命名：
- 电影：`电影名.年份.分辨率.媒介.视频编码.音频编码-发布小组名`
- 剧集：`剧集名.季/季集.年份.分辨率.(平台).媒介.视频编码.音频编码-发布小组名`
- 发布完成后站点自动去除标题中的点（保留音轨声道中的点）。

### 2.3 禁止资源

- 标清 upscale 视频
- 劣质视频：CAM/TC/TS/SCR/DVDSCR/R5/R5.Line/HalfCD
- DivX/XviD（.AVI）、RealVideo（.RM/.RMVB）、Flash
- AI 算法插帧制作的不符源帧率资源
- 禁忌/敏感内容
- **黑名单制作组**：FGT、Hao4K、Mp4Ba、RARBG、Mooko
- 无法声明来源或疑似转自公网的资源
- 单碟 BDMV 结构不完整/拆分发布

### 2.4 合集打包规则

- 原则上只允许以下资源打包：
  - 按套装售卖的高清电影合集
  - 整季电视剧/综艺/动漫
  - 同一专题的纪录片
  - 分卷发售的动漫剧集、角色歌、广播剧
- **合集只允许发布 Untouched 原盘**（2023.04.20 更新）
- 原盘 DIY/Remux/Encode/WEB-DL 不再允许发布合集
- 同一 IMDB/豆瓣号下正片多分辨率打包禁止

### 2.5 综艺类特殊要求

综艺类资源必须满足至少一项：国语配音或中文字幕（可外挂），否则禁止发布。

### 2.6 已完结剧集

- 已完结剧集的分集资源不可单独发布（只可发布完结合集）。
- Web-DL/HDTV/WebRip 媒介的剧集单季完结打包资源必须勾选"完结"标签。

## 三、Trump（取优）及共存规则

> 来源：论坛帖 `forums.php?action=viewtopic&forumid=7&topicid=4858`

### 3.1 取优原则（Trump）

同分辨率下：
1. 视频编码一致 + 音频编码一致 + 视频码率高于现存 **20%** → 可 trump
2. 视频编码一致 + 音频编码/码率更优（如 AAC→DD/DDP/DTS）+ 视频码率不低于现存 **90%** → 可 trump
3. 编码/码率一致 + 额外加入国语/母语配音 → 可 trump
4. WebRip（源自 Web-DL 的二次压制）→ 如站内唯一版本保留；如后发布 Web-DL 版且无更优素材 → 可被 trump

### 3.2 共存原则

1. 视频码率 < 现存 80% + 音频更优 → 共存
2. 有原生中字 + 现存无中字 → 共存
3. 有原生国语配音 + 现存无国语配音 → 共存
4. 原生平台小体积高分辨率 + 现存大体积低分辨率 → 共存（4K 只保留 H265，1080p 保留 H264）
5. 审查删减/加长版/导演剪辑版/画幅不同 → 共存
6. 非源码被源码 trump 时，源码作为特殊版本共存
7. 不同 OTT 平台无中字版 + 视频码率高于现存 50% → 共存

### 3.3 保护制作共存

非平台发布的中字版本（后封制作）→ 可与现存无中字版本共存。后续版本需在评论说明素材及截图。

### 3.4 关键规则

- **SDR/HDR/DV 作为三个独立版本**，可共存。
- MKV 容器可 trump MP4 容器。
- 站内官组/转载组资源被 trump 后仍保留。
- 已完结的分集官组资源未打包前不可被 trump（除非新资源可 trump 预期打包后资源）。

## 四、发种 H&R 规则

> 来源：论坛帖 `forums.php?action=viewtopic&forumid=7&topicid=5166`（2025-05-01 实行）

- 发布者需在 **7 天内做种 48 小时**，无论是否出种。
- 审核错误未修改种子达到 2 个 → 暂停发布权限。

---

## 五、盒子规则

- 盒子下载流量按 100% 计算，不享受任何优惠（黑种）。
- 官方种子发布后 72 小时内，盒子上传流量最多统计为汇报下载量的 **3 倍**。
- 单个种子统计上传量超过种子体积 **3 倍**后，盒子上传不再统计。
- VIP 用户组和工作组不受限制。
- 阻断列表：Azure/Google Cloud/AWS/Oracle Cloud/DigitalOcean/Linode/Vultr。

---

## 六、其他规则

### 6.1 下载 H&R

- 对贵宾等级之外所有成员生效。
- H&R 对发布超过 30 天的种子不生效。
- 有 H&R 的种子 14 天内须做种 48 小时或分享率达 1.0。
- 累计 10 次 H&R 警告 → 封禁。消除 1 次警告需 50000 爆米花。

### 6.2 爆米花系统

- 发布新种子：300 爆米花（<1G 种子仅 100）。
- 上传字幕：20 爆米花。
- 可用于置顶促销种子。

### 6.3 禁转规则

- 官方发布组资源禁止转载至 **PTTime**（2022/10/13 起）。
- 违反禁转规则将封禁账户，邀请人连带处罚。

### 6.4 连带惩罚

- 交易/作弊/多号 → 树下全部用户连带封禁。
- 下家威胁站点/辱骂管理/违规禁转 → 邀请人永久禁止邀请权。

### 6.5 超速封禁

- 单种上传速度超过 **125MB/s** → 自动封禁。
- 第一次可解封（需提供 IP 测速视频），第二次永久封禁。

## 七、适配器设计注意事项

### 7.1 字段映射特点

| 特点 | 说明 |
|------|------|
| 无制作组下拉 | 制作组信息从标题 0day 命名中的 `-Group` 部分提取 |
| 无来源/地区字段 | 不需要映射 `source_sel`、`processing_sel` |
| 豆瓣 ID 字段 | `douban_id` 可接受 ID 或完整链接 |
| HDR 三标签 | db(hdr10/hdrm) 需根据 MediaInfo 准确区分 HDR10/HDR10+/DV |
| None 分辨率 | 音乐/电子书等非视频资源使用 `standard_sel=11` |
| 完结标签强制 | Web-DL/HDTV 剧集完结打包必须勾选 `wj` |

### 7.2 黑名单制作组

以下制作组资源禁止发布：**FGT、Hao4K、Mp4Ba、RARBG、Mooko**。

### 7.3 Trump 规则适配

发布前需检查站内同 IMDB/豆瓣号下是否已有资源，按 Trump 及共存规则判断：
- 视频码率差异阈值：20%（取优）、80%（共存）
- SDR/HDR/DV 三版本独立
- MKV 优先于 MP4

### 7.4 Cloudflare 防护

需使用 utls + rod 降级方案绕过 TLS 指纹检测（参见 `docs/31-模块设计决策记录.md §29`）。
