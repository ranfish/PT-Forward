## 一、发布页面表单字段分析

**提交地址**: `takeupload.php`（POST multipart/form-data）

**字段名无后缀**（裸名，如 `medium_sel` 而非 `medium_sel[4]`）。

**发布流程**: 非官方组用户需先在候选区提交候选，审核通过后方可发布。

### 1.1 基础信息字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| `file` | file | ✓ | 种子文件 |
| `name` | text | - | 标题（0day 命名法，发布后自动去除点号） |
| `small_descr` | text | - | 副标题（中文名、字幕信息、音轨信息等） |
| `douban_id` | text | - | 豆瓣 ID/链接（可输入豆瓣影视 ID 或完整链接） |
| `descr` | textarea | ✓ | 简介（BBCode，需含豆瓣简介 + MediaInfo + 截图） |
| `uplver` | checkbox | - | 匿名发布（值=yes） |
| `hr` | checkbox | - | HR 标记（值=1） |
| `waitcheck` | checkbox | - | 等待审核（值=1） |
| `banned` | checkbox | - | 禁转标记（值=yes） |
| `url` | text | - | IMDb 链接（如 `http://www.imdb.com/title/tt0468569/`） |
| `nfo` | file | - | NFO 文件（不允许 User 以下用户查看） |

- **无 PT-Gen 字段**，需手动在 `descr` 中填入豆瓣信息。

- **无 `source_sel`、`processing_sel`、`team_sel` 字段**——这是 PTerClub 的显著特征。
- 原盘类资源必须包含 BDInfo，其他资源须包含 MediaInfo。
- 标题命名规范：`电影名.年份.分辨率.媒介.视频编码.音频编码-发布小组名`。

## 二、发布规则要点

### 2.1 候选制

- 非官方组用户需先在候选区提交候选。
- 候选区当前有 7 个待审项（导航栏显示"候选(7)"）。

### 2.2 标题命名（0day 命名法）

影视资源必须使用 0day 命名：
- 电影：`电影名.年份.分辨率.媒介.视频编码.音频编码-发布小组名`
- 剧集：`剧集名.季/季集.年份.分辨率.(平台).媒介.视频编码.音频编码-发布小组名`
- 发布完成后站点自动去除标题中的点（保留音轨声道中的点）。

### 2.3 禁止资源

- 标清 upscale 视频
- 劣质视频：CAM/TC/TS/SCR/DVDSCR/R5/R5.Line/HalfCD
- DivX/XviD（.AVI）、RealVideo（.RM/.RMVB）、Flash
- AI 算法插帧制作的不符源帧率资源
- 禁忌/敏感内容
- **黑名单制作组**：FGT、Hao4K、Mp4Ba、RARBG、Mooko
- 无法声明来源或疑似转自公网的资源
- 单碟 BDMV 结构不完整/拆分发布

### 2.4 合集打包规则

- 原则上只允许以下资源打包：
  - 按套装售卖的高清电影合集
  - 整季电视剧/综艺/动漫
  - 同一专题的纪录片
  - 分卷发售的动漫剧集、角色歌、广播剧
- **合集只允许发布 Untouched 原盘**（2023.04.20 更新）
- 原盘 DIY/Remux/Encode/WEB-DL 不再允许发布合集
- 同一 IMDB/豆瓣号下正片多分辨率打包禁止

### 2.5 综艺类特殊要求

综艺类资源必须满足至少一项：国语配音或中文字幕（可外挂），否则禁止发布。

### 2.6 已完结剧集

- 已完结剧集的分集资源不可单独发布（只可发布完结合集）。
- Web-DL/HDTV/WebRip 媒介的剧集单季完结打包资源必须勾选"完结"标签。

## 三、媒介/标签/编码选择释义

> 来源：论坛帖 `forums.php?action=viewtopic&forumid=10&topicid=686`（已锁定）

### 3.1 媒介分类详解

| 媒介 | 说明 |
|------|------|
| UHD Blu-ray | 4K 蓝光原盘，仅经实体盘破解/解除锁区，无任何改动（Untouched）。发布时注意是否 HDR/DV，需勾选相应标签 |
| UHD BluRay/DIY | 对 4K 蓝光原盘进行自定义操作：添加/删减音轨、字幕轨、修改菜单、加入导评及花絮字幕等 |
| Blu-ray 原盘 | 1080P/1080i 蓝光原盘，仅经实体盘破解/解除锁区，无任何改动（Untouched） |
| BluRay/DIY | 对 1080P 蓝光原盘自定义操作。miniBD 资源虽主视频经重编码，但具备蓝光结构，归为此类 |
| Remux | 对 UHD/BD 原盘正片或花絮，删除或增加音轨、字幕等，单独制作 mkv/mp4 文件，视频与主音轨质量等同原盘 |
| Encode | 对原始视频资源进行有损重编码（压制），标题多见 x264/x265/Rip 字样 |
| Web-DL | 直接从流媒体平台下载的资源（如 Netflix），封装为 mp4/mkv |
| HDTV | 采用特殊设备提取的数字源高清电视资源，UHDTV 可能有 HDR |
| DVD | DVD 原盘，最高分辨率 720p |
| 其他 | 以上四类原盘/DIY 需具备完整蓝光结构文件夹或打包为 ISO；Remux/Encode/Web-DL 为独立 mkv/mp4 文件 |

### 3.2 标签使用详解

| 标签 | 完整释义 |
|------|----------|
| 原创 | 自己购买发布的原盘、自己 DIY/Encode/Remux、自己从流媒体平台下载的资源，包括自获取的音乐、有声书等 |
| 国语/粤语 | 分别指国语/粤语配音。正片只有一条混合语言音轨时，根据资源主要语言确定标签 |
| 官字组 | 本站官方字幕组出品介入制作的资源，仅适用于 Audies 官组或本站字幕组帮助制作的资源。2023-05-23 补充：并非全部制作流程由 DIY 组完成，部分仅为样式调整和细节校准 |
| 中字 | 简体/繁体字幕，或带有简繁双语字幕。包括内封软字幕、硬字幕 |
| Dolby Vision | 杜比视界资源，原盘双视频轨道或制作后 MEL（单层）/FEL（双层）视频轨道。DV 资源一定包含 HDR。蓝光 DV 截图与 HDR 相同，流媒体 DV 可能截图发绿/发紫 |
| HDR10 | 含 HDR10 标准，多见于 MediaInfo：BT2020 + HDR10 compatible |
| HDR10+ | 含三星 HDR10+ 标准，多见于 MediaInfo：BT2020 + HDR10+ compatible，较少见 |
| 禁转 | 禁止转载他站的资源。一般不适用于从友站转载来到本站的资源。**管理确认：禁转针对所有站点，不仅仅是 PTTime** |
| 限转 | 限时禁止转载，标签消失前不可转载 |
| DiY | 仅适用于原盘资源的 DiY 制作 |
| 首发 | 仅适用于蓝光原盘的首次全网发布（不仅是本站） |
| 应求 | 应求种区需求而发布 |
| 动画 | 非真人的动画/动漫类视频资源 |
| 完结 | 剧集类已完结的整季发布资源，Web-DL/HDTV/WebRip 媒介必选 |

**HDR 说明**：
- HDR 资源必须具备 BT2020 色域及动态亮度信息。
- 即使是 4K 资源，如不具备这两个指标仍非 HDR（如老片修复的 4K BT709 原盘、大部分国内流媒体 4K 资源）。
- HDR10 = ST2086 静态元数据；HDR10+ = ST2094 动态元数据。
- HLG 标准 HDR 资源归类至 HDR10。

## 四、Trump（取优）及共存规则

> 来源：论坛帖 `forums.php?action=viewtopic&forumid=7&topicid=4858`

### 4.1 取优原则（Trump）

同分辨率下：
1. 视频编码一致 + 音频编码一致 + 视频码率高于现存 **20%** → 可 trump
2. 视频编码一致 + 音频编码/码率更优（如 AAC→DD/DDP/DTS）+ 视频码率不低于现存 **90%** → 可 trump
3. 编码/码率一致 + 额外加入国语/母语配音 → 可 trump
4. WebRip（源自 Web-DL 的二次压制）→ 如站内唯一版本保留；如后发布 Web-DL 版且无更优素材 → 可被 trump

### 4.2 共存原则

1. 视频码率 < 现存 80% + 音频更优 → 共存
2. 有原生中字 + 现存无中字 → 共存
3. 有原生国语配音 + 现存无国语配音 → 共存
4. 原生平台小体积高分辨率 + 现存大体积低分辨率 → 共存（4K 只保留 H265，1080p 保留 H264）
5. 审查删减/加长版/导演剪辑版/画幅不同 → 共存
6. 非源码被源码 trump 时，源码作为特殊版本共存
7. 不同 OTT 平台无中字版 + 视频码率高于现存 50% → 共存

### 4.3 保护制作共存

非平台发布的中字版本（后封制作）→ 可与现存无中字版本共存。后续版本需在评论说明素材及截图。

### 4.4 关键规则

- **SDR/HDR/DV 作为三个独立版本**，可共存。
- MKV 容器可 trump MP4 容器。
- 站内官组/转载组资源被 trump 后仍保留。
- 已完结的分集官组资源未打包前不可被 trump（除非新资源可 trump 预期打包后资源）。

## 五、站点完整规则

### 5.1 总则

- 作弊
- 恶意限速做种
- 恶意不尊重上传者（去除后缀/违反禁转/修改文件/夹带广告病毒等）
- 以站点资源盈利
- 将站点资源公开分享至百度/OD/GD/115 等网盘、BT 网站等
- 使用未经允许或自行编写的工具爬取站点数据，或请求频率超过正常范围（允许的工具：PTPP/MP/VT/IYUU/Ptee 及转种脚本）

### 5.2 账号保留规则

### 5.3 上传总则

- 上传者必须对上传的文件拥有合法的传播权。

- 用户需要先在候选区提交候选。
- 非官组用户必须走候选流程，审核通过后方可发布。
- 用户种子合规积分为负值时，种子将强制发布到候选区。

### 5.5 发种 H&R 规则

> 来源：论坛帖 `topicid=5166`（2025-05-01 实行）+ `rules.php`

- H&R 制度对发布员等级之外的所有成员生效。
- **发布者需要在 7 天内做种 48 小时**，无论是否出种。
- **用户特别说明：本站有发种 72 小时 HR 规则，即发种必须保证 72 小时做种。**
- 审核错误未修改种子达到 2 个 → 暂停发布权限。
- 发布信息多次填写错误/不规范，发布者可能会被警告、强制候选甚至取消上传权限。

### 5.6 下载 H&R 规则

- H&R 制度对贵宾等级之外的所有成员生效。
- H&R 对发布超过 30 天的种子不生效。
- H&R 记录在种子首次下载完成时产生。
- 有 H&R 的种子 **14 天内**必须做种 **48 小时**或分享率达 **1.0**。
- 累计 **10 次** H&R 警告 → 账号立即封禁。
- 消除 1 次 H&R 警告需花费 **50000 爆米花**。
- 提示：本站很少有种子存在 HR，有 HR 的种子会显示黑色标签。

### 5.7 种子促销规则

- 随机促销：种子上传后系统自动随机设为促销。
- 关注度高的种子将被设置为促销（由管理员定酌）。
- 不定期开启全站免费。
- 每小时最多允许下载 200 个种子，超限自动禁止下载 4 小时。

### 5.8 爆米花系统

**获取方式**：
- 发布新种子：300 爆米花（<1G 种子仅 100）。
- 上传字幕：20 爆米花。

**置顶促销规则**：
1. 用户通过支付爆米花来改变种子的置顶位置和促销类型。
2. 种子置顶每小时消耗 **100 爆米花**。
3. 已置顶种子可通过追加竞价爆米花改变排列顺序。
4. 促销消耗：
   - **Free**：10G 及以下体积基础 200 粒/小时；10G 以上按公式增加。
   - **2X Free**：10G 及以下体积基础 400 粒/小时；10G 以上按公式增加；**80G 以上无法促销为 2X Free**。
5. 促销类型修改后覆盖之前所有类型。
6. 管理员可改变置顶和促销类型，已支付爆米花不退还。

**消耗计算公式**：
```
倍率 = 1 + (种子大小G - 10) / 10 × 0.1
实际消耗 = 基础值 × 倍率
```

### 5.9 盒子规则

**定义及标记**：
- 在独立服务器/VPS 上安装客户端软件后上传下载。
- 任何非家庭宽带 IP 环境。
- 使用代理软件全局模式/大陆白名单模式未将 tracker 地址（`t.audiences.me` 及 `tracker.cinefiles.info`）添加例外 → 会被标记盒子。

**规则**：
- **下载**：盒子下载流量按 100% 计算，不享受任何种子优惠（黑种）。
- **上传**：
  - 官方种子发布后 **72 小时内**，盒子上传流量最多统计为当次汇报下载量的 **3 倍**。
  - 单个种子统计上传量超过种子总体积 **3 倍**后，盒子上传不再统计。
- **例外**：VIP 用户组和工作组不受此限制。

**阻断列表**（2022-05-01 起）：Azure / Google Cloud / AWS / Oracle Cloud / DigitalOcean / Linode / Vultr。

**2023-05-10 更新**：如无重要规则更新，不再更改站点代码。试图逃避盒子规则的行为将直接封禁。

## 六、适配器设计注意事项

### 6.1 字段映射特点

| 特点 | 说明 |
|------|------|

### 6.3 Trump 规则适配

- SDR/HDR/DV 三版本独立
- MKV 优先于 MP4

需使用 utls + rod 降级方案绕过 TLS 指纹检测（参见 `docs/31-模块设计决策记录.md §29`）。
