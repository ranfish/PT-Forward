## 一、发种规范（来自论坛 topicid=7984）

### 1.1 总则

- 英简繁规则有出入的地方，以简体中文规则为准
- 发布涉及政治、三俗或广告交易等危害论坛发展的信息的用户，管理员有权直接封禁
- 请尊重发布者。任何不尊重发布者的言论一经核实，管理员有权直接封禁
- 严禁任何形式的账号交易行为（魔力值交换、邀请码交换、账号交换、贩卖或变相贩卖等）
- 禁止求魔
- 上传本站的种子文件到其他 Tracker 的用户，管理员有权直接封禁

### 1.2 账号保留规则

| 等级 | 保留条件 |
|------|----------|
| 贵嫔-正三品(Veteran User) 及以上 | 永远保留 |
| 贵人-正六品(Elite User) 及以上 | 封存账号后不会被删除 |
| 常在-正七品(Power User) 及以下 | 封存账号后连续 **180天** 不登录将被删除 |
| 未封存账号 | 连续 **90天** 不登录将被删除 |
| 无流量用户（上传/下载都为0） | 连续 **15天** 不登录将被删除 |

### 1.3 上传总则

- 任何人都能发布资源
- 上传者必须对上传的文件拥有被允许的传播权
- 发布者必须在做种累计时间达到 **24 小时**且有至少 **3 个做种者**后方可撤种，若 **72 小时**后无人下载可撤种
- 发布者将获得 **双倍上传量**
- 种子及种子简介不得包含与资源信息无关的网站链接、广告文档等
- 违规资源不经提醒直接删除

### 1.4 上传者资格与候选制

- 任何等级发布种子数 **≤3** 时，都会**强制候选**
- 正六品(Elite User) 及以上等级发布种子数 **>3** 个时，可直接发布种子
- 正七品(Power User) 及以下等级需先在候选区提交候选

### 1.5 允许的资源与文件

种子仅允许：
- 官方正式发行的专辑及合集
- 商业销售的实体专辑
- 通过流媒体平台发布的专辑、单曲

#### 1.5.1 CD 资源

- 由抓轨设备从 CD 上抓取的包含 log 文件的音乐资源，规格仅为 **16bit/44.1kHz**
- 允许格式仅为 **FLAC 分轨**（单文件只包含一首歌曲）和 **WAV 整轨**（单文件包含整张专辑）
- 发布必须提供封面图片，不要求提供频谱图

#### 1.5.2 Web 资源

- 下载于网络音乐平台、媒体或销售的数字专辑，及**不包含 log 的 CD 资源**
- 采样频率达到 **44.1kHz** 及以上且比特深度达到 **16bit** 及以上，但不超过 **24bit/88.2kHz**
- 资源类型包含 FLAC 分轨、WAV 整轨
- **MQA 制式资源以展开前规格作为资源规格**
- 发布必须提供封面图片，要求提供频谱图

#### 1.5.3 Hi-res 资源

- 采样频率达到 **88.2kHz** 及以上且比特深度达到 **24bit** 及以上
- 上传资源时必须附上封面图片及频谱图，频谱图必须根据资源的规格达到相应的频率范围（如 24bit/88.2kHz 的资源，频谱图刻度必须达到 44.1kHz，以此类推）
- 资源分类必须选择 **Hi-res**
- 音频格式原则上仅允许 **FLAC 分轨**，如为其他格式必须为官方发行原始格式并在资源介绍页面附上官方发行证明

#### 1.5.4 SACD、DSD 资源

- 仅允许由官方发售的原生采用 DSD 编码技术的资源，包含实体 SACD 光碟、DFF 文件、DSF 文件
- 原则上仅允许以 **SACD 镜像整轨**、**DFF 分轨**、**DSF 分轨**的形式发布
- SACD 镜像整轨发布时格式选择镜像，DSD 编码分轨发布时格式选择对应的格式
- 发布必须提供封面图片，不要求提供频谱图

#### 1.5.5 DTS 资源

- 由官方发行的多音轨（≥6 条音轨）并使用 DTS 编码的资源
- 如 DTS 资源原载体为 Blu-ray Audio Disc，资源必须包含 **BDinfo 信息**作为 log 上传在介绍页面，内容无任何保护和限制，**强制候选**
- DTS 资源仅允许 **ISO 镜像整轨**、**DTS 分轨**
- 发布必须提供封面图片，要求提供频谱图

#### 1.5.6 转录资源

- 限定为"非数字化存储的音频文件进行转录所得的数字化音频文件"
- 规格下限为 **16bit/44.1kHz**，上限为 **24bit/96kHz**
- 必须携带相应的**设备 LOG** 进行上传，否则不被视为转录资源，将视作 Web 资源
- 发布必须提供封面图片，要求提供频谱图

#### 1.5.7 Other 资源

- 任何除以上规定以外的数字化音乐文件
- **强制提交候选**，等待管理审核

### 1.6 不允许的资源与文件

**不允许的资源类型**：
- 用户个人编辑的作品（商业上正式发售的除外），包括但不限于：从专辑中选取不同曲目整合而成的作品、根据某个音乐榜单制作的专辑、混音作品
- 从电影、MV 等视频中提取的音频
- 从游戏文件中提取的音频（允许官方游戏光盘或数据包中独立存在的音频文件，需出示证据）
- 电影和视频
- 演唱会视频
- 音乐影片

**不允许的文件**：
- 总体积小于 **50MB**（单曲专辑除外）
- 有损音频资源（MP3 等需移步外站）
- **APE** 音频文件（明确禁止）
- **TTA** 音频文件
- CD 镜像
- **rerip** 资源
- 相关音乐类软件或书籍
- 无正确 cue 表单的整轨音频文件
- 压缩文件（RAR/ZIP 等）
- 损坏的文件、垃圾文件、涉及禁忌或敏感内容的资源

### 1.7 重复、替换、共存规则（Dupe）

根据资源类别及资源性质划分为 **6 大类**，每种类之间重复、替换、共存规则相互独立。

**OpenCD 制作组和 LLM 组原抓不受此规则限制。**

#### 1.7.1 CD 资源 Dupe 规则

- 包含 LOG 文件并进行评分的资源才可以判定为 CD 资源
- 同一专辑名称的不同发行版本视为不同资源，不构成重复
- CD 资源的质量以 **LOG 质量**为标准
- 无论何种格式，**高分 LOG 替换低分 LOG**
- 当且仅当同一专辑的两个不同格式的资源的 LOG 都是满分时，这两个资源（其一为 FLAC，另一个为 WAV）才可以共存。相同格式的资源不能共存。同质量资源的 WAV 和 FLAC 格式视为重复种
- 满分 LOG 的资源因为缺少 noncompliant cue 可以被含有 noncompliant cue 的满分 LOG 的资源替换
- 在已存在满分 LOG 的转载资源的情况下，后上传的 **OpenCD 和 LLM 原抓组**的资源不算重复。反之，已存在原抓组资源的情况下，同质量的资源算作重复种，不允许上传
- 由于音质无关的选项而被扣分的 LOG 可以替换音质相关项减分的 LOG。一个缺少 checksum 的 LOG 可以替换 offset 不正确的 LOG。相反只能 100 分才能替换前者

#### 1.7.2 WEB 资源 Dupe 规则

- 经流媒体或其他非实体途径发行的音乐资源为 WEB 资源，并同时包含**无 log 或 0 分 log 的 CD 资源**
- 替换规则以**高规格替换低规格**为主：采样深度高的替换采样深度低的，相同采样深度下以采样频率高的替换采样频率低的
- WEB 资源最高规格为 **24bits/48kHz**

#### 1.7.3 Hi-res 资源 Dupe 规则

- 以**高规格替换低规格**为主：新资源的采样频率和比特深度中至少要有一项比老资源高且不得有任何一项比老资源低（如 24bit/192kHz 可替换 24bit/96kHz，但不允许 24bit/192kHz 替换 32bit/96kHz）
- 需要进行替换的资源，必须手动提交候选，并在副标题注明"**申请替换**"
- 进行资源替换时，新资源必须提供官方发行证明，或任何可以证明其资源出自于官方的证明，并附在资源介绍中

#### 1.7.4 DSD 资源 Dupe 规则

- 原则上不允许进行替换，除非出现资源质量问题可以单独申诉进行替换
- **DFF 和 DSF 格式**可以有 DSD64、DSD128、DSD256 三种不同规格的文件共存（必须在副标题注明），但**不允许不同文件格式相同采样率的文件共存**

#### 1.7.5 DTS 资源 Dupe 规则

- 原则上不允许进行替换，除非出现资源质量问题可以单独申诉进行替换
- 需要进行替换的资源，必须手动提交候选，并在副标题注明"**申请替换**"

#### 1.7.6 转录资源 Dupe 规则

- 对同一个艺术家的同一张专辑的同一种硬件媒介类型仅允许一个资源存在，否则视为重复
- 对于同一专辑，**高规格替换低规格**
- 同一专辑同一规格原则上不允许进行替换，除非出现资源质量问题可以单独申诉进行替换

#### 1.7.7 Dupe 总体须知

- 严禁编辑修改 LOG 文件内容
- 上传者如被发现故意伪造质量来恶意替换资源将会受到严肃处理
- 使用 rerip 资源冒充原抓恶意替换资源会受到严肃处理
- 扫图不作为质量判定依据
- 最高质量的种子如果违反规则或者存在瑕疵则可以被符合规则的资源替换，替换资源请添加至候选并说明情况、提供证据
- 未在资源页面上传 LOG 或 0 分 LOG 的 CD 类资源会被视为"没有 LOG"，列入 Web 类资源
- 如果明确是 scene 资源，请注明制作组
- 种子发布**一周后**，除资源质量问题，任何人（包括发布者本人）不得删种

### 1.8 老种处理规则

- 符合新规则的老种将保留
- 非官方专辑将被删除。非官方合集当且仅当其包括的专辑都有相应资源时被删除
- 同一资源的多个老种依照以下规则处理：
  - 早于原抓组作品的满分 LOG 种子不算为重复
  - **满分 LOG 的老种将永久保留**（即使抓轨 LOG 形式（分轨/整轨）和文件形式不一致）
  - 特例：当且仅当存在 LOG 与其相同的老种，则较晚发布的种子将会被删除
  - 其他情况以"**质量优先**，**上传时间其次**"的优先级顺序筛除：若两个种子质量相同，则上传时间较晚的那个种子将被删除
  - 和老种相同 LOG 的新种不允许上传，除非老种质量上存在瑕疵
- 其余不符合新规则的种子将保留直到符合新规则的新种被上传
- APE 替换照旧，处理方案不变

### 1.9 标题命名规则

标题命名必须以官方发行信息相同。严禁任何别名、译名。

- 中国大陆发行的专辑如果官方以中文译名发行，则专辑名可以使用官方的中文译名（参照封面和发行信息）
- 不同版本的资源必须是完全不同的 CD。相同 CD 即使不同包装、不同国家地区发行，视为重复种
- 标题的符号只允许使用**半角**
- 单曲型 FLAC 类资源分类为 FLAC 分轨，单曲型 WAV 类资源分类为 WAV 整轨

**标题格式**（由 JS 函数 `titlepreview()` 自动生成）：
```
艺术家 - 专辑名称 年份 - 格式 [整轨|分轨] - 原创者 - 来源
```
古典音乐特殊格式：
```
艺术家 - 作曲家 - 专辑名（年代） - 格式 - 作者/来源
```

范例：
- `Jerry Goldsmith - Total Recall [The Deluxe Edition] 2000 - FLAC - barca008@OpenCD`
- `Claudio Abbado & Berliner Philharmoniker - Mahler: Symphony no. 6 in A minor 2005 - FLAC - 小胖@OpenCD`

字段拼接顺序：artist（` - `）、resource_name（` - `）、year（` `）、standard、audio_mode、pgname（` - `）、frname（` - `）

### 1.10 简介规则

- 专辑或合集发布需要包含：**专辑封面**、**专辑简介**和**曲目列表**、**频谱图**（有 log 不强制，否则需要贴整个专辑的频谱图）
- 抓包 LOG 信息应包含至发布文件中，同时上传至 NFO/LOG
- 对于官方合集，已有合集种子，则单 CD 种子将被删除
- DTS 音频资源必须贴出频谱图
- Blu-ray Audio 资源需附 BDinfo 信息和官方发行信息

### 1.11 转载资源文件规则

- 除特殊情况，资源必须包含封面图一张，且命名为 Front 或 Cover.jpg/png
- Web 发行的资源必须与原始发行形式吻合（分轨/整轨）
- DTS 音频资源必须贴出频谱图，DTS 资源的音频必须是 DTS 编码，不允许其他编码的 DTS 资源
- 资源文件必须体现资源描述：
  - 文件夹名至少包含专辑名，建议使用"艺人 - 专辑名(年代)[格式]"
  - 特殊版本信息需明确标注（deluxe edition、remastered 以及年份、EMI/DECCA、Japanese Edition 等）
  - FLAC 文件至少包含音轨号和名称（如 `01.abc.flac`），WAV 整轨文件名必须至少包含专辑名字
  - 套装(boxset)允许以碟号命名（如 `Disc 1.wav`），但不允许同一目录下出现相同音轨号的多个文件
- 如果带有子文件夹，文件夹名必须体现其内部内容（如 `bonus disc`、`CD1`、`Artwork`、`scans`）
- 所有文件的文件名必须体现其内容或作用（不允许如 `QQ截图xx.png` 的文件）
- FLAC 文件必须正确打上标签(tag)，至少包含：音轨号(Tracker Number)、艺人(Artist Name)和专辑名(Album Name)。WAV 文件对应的 cue 必须至少写明艺人和专辑名
- 文件任何地方不允许存在广告性质和第三方无关内容（包括文件夹名、FLAC tag、cue 内容、文件名、图片水印等）
- 除特殊情况，附带的 DVD(Bonus DVD) 必须是 ISO 形式且必须是原盘
- 纯文本文件必须是 UTF-8 编码

### 1.12 新手考核

新人注册起第一个月必须满足（否则 ban）：
- 上传 > 10G
- 下载 > 10G
- 魔力值 > 1000
- 分享率 > 1
- 做种率 > 6.0

## 二、H&R 规则（Hit & Run）

> 来源：rules.php + 论坛 topicid=8520（H&R 规则详解）

### 2.1 基本规则

- 采取**全局 H&R 制度**，即所有种子都计算 H&R 次数
- **计算方式**：种子下载完成后，该种需在 **30 天内做种 36 小时**，未达到此要求则记一次 H&R
- H&R 满 **10 次**，即 ban
- **皇后及以上级别（包括 VIP）**：计算 HR 次数，但**不 ban 号**
- 新人注册一个月内：计算 HR 次数但不 ban 号；第二月起满次数即 ban

### 2.2 H&R 消除方式

| 方式 | 说明 |
|------|------|
| 自动消除 | 获得该 H&R 的种子在 **90 天内**达到 **300 小时**做种时间，自动消除该 H&R |
| 魔力值兑换 | **10,000 魔力值**消除一次 H&R 次数 |

### 2.3 豁免

- **发种者**与**辅种者**不计算该种子的 H&R 次数
- 正五品及以上等级由于 H&R 被 ban 可在外站申请开通，但需接受考核：H&R 次数改为 5、魔力值清零、接受通用新手考核

## 三、种子促销规则

### 3.1 随机促销（种子上传后系统自动随机设定）

| 概率 | 促销类型 |
|------|----------|
| 10% | 50% 下载 |
| 10% | 30% 下载 |
| 10% | 免费(FREE) |
| 5% | 2x 上传 |
| 5% | 50% 下载 & 2x 上传 |
| 0% | 免费 & 2x 上传(2XFREE) |

### 3.2 自动促销

| 条件 | 促销类型 |
|------|----------|
| 文件总体积 **>10GB** | 自动成为 **免费(FREE)** |
| 发布一个月后做种人数 **<3** | 自动设置为 **FREE** |
| 发布一个月后做种人数 **=0** | 自动设置为 **2XFREE** |
| 关注度高 | 管理员手动设置促销 |

### 3.3 促销时限

- 如果离发布时间已过去 **3 天**，所有随机促销将变为普通

## 四、转载规则（4 级）

> 来源：rules.php 转载规则章节

| 等级 | 标注语示例 | 转载要求 |
|------|-----------|----------|
| **禁止转载** | "谢绝任何形式的转载" | 不可转载该发布者的任何资源 |
| **未经允许不得转载** | "未经允许请勿转载到其他论坛" | 需先询问发布者本人，经允许后方可转载 |
| **欢迎转载** | "欢迎转载各大PT论坛，请在帖子顶部注明出处及作者，并保留后缀" | 可直接转载，但必须在帖子顶部注明出处及作者，并保留后缀 |
| **其他情况** | 参考副标题及种子简介的标注语 | 视具体情况而定 |

**转载总则**：
- 无论如何，请尊重发布者
- 严禁违规转载 OpenCD 原抓资源，一经发现即 BAN 号处置

## 五、发布页面表单字段分析

## 八、关键适配器设计要点

### 8.1 标题自动生成

音乐类（type=408）的标题由前端 JS 自动拼接，字段顺序：
```
{artist} - {resource_name} {year} - {standard名称} {audio_mode名称} - {pgname} - {frname}
```

适配器需要 **在后端重现此逻辑**，生成正确格式的标题。

### 8.2 候选制发布

所有种子默认以候选状态发布（`status=pending`，disabled 且 checked），需要通过投票审核后才能正式出现在种子列表。**Elite User 及以上**等级可直接发布无需候选。

### 8.3 频谱图/Log 校验

提交前校验逻辑：
- 有 Log → 直接通过
- 无 Log 且格式为镜像(1)/DFF(17)/DSF(18) → 通过
- 无 Log 且其他格式 → **必须上传频谱图**

适配器需根据格式类型判断是否需要频谱图。

### 8.4 特殊字段名差异

| 通用 NexusPHP 字段 | OpenCD 字段 | 说明 |
|--------------------|-------------|------|
| `torrent`/`torrentfile` | `file` | 种子文件字段名不同 |
| `upload.php` | `plugin_upload.php` | 发布页面路径不同 |
| `takeupload.php` | `plugin_upload_save.php` | 提交地址不同 |
| 无 | `source` | 音乐流派（其他站用作 source_sel） |
| 无 | `standard` | 音频格式 |
| 无 | `medium` | 物理媒介 |
| 无 | `audio_mode` | 整轨/分轨 |
| 无 | `artist` | 艺术家 |
| 无 | `resource_name` | 专辑名 |
| 无 | `track_list` | 曲目列表 |
| 无 | `audition` | 试听音频 |
| 无 | `spectrogram` | 频谱图 |
| 无 | `share_rule` | 转载规则 |
| 无 | `boardid`/`frname`/`pgname` | 来源/原创者信息 |
| 无 | `cover` | 封面（独立字段，非描述内） |

### 8.5 重复检测 API

```
GET plugin_upload.php?cmd=checkTorrent&search=<资源名>
POST rows=100
```

返回 JSON：`{"total": N, "rows": [{"id": "...", "name": "..."}]}`

适配器可在发布前调用此 API 检查重复。

### 8.6 标签 API

```
GET plugin_upload.php?cmd=gettags
```

返回 JSON：`{"items": [{"id": "1", "name": "大陆", "parentId": "1", "parentName": "地区(Area)"}, ...]}`

### 8.7 用户等级与发布权限

- **Elite User 及以上**（正六品）：无需候选，可直接发布
- **Power User 及以下**：需先提交候选，通过投票后正式发布
- **发布员(Uploader)**：免除自动降级，可查看匿名用户

## 十、特殊注意事项

### 10.1 发布页面路径

OpenCD 使用 **定制插件发布页** `plugin_upload.php`，而非标准 NexusPHP 的 `upload.php`。提交地址为 `plugin_upload_save.php`，也非标准的 `takeupload.php`。适配器必须使用正确的路径。

### 10.2 封面上传

封面通过 `attachment.php?target=cover` iframe 上传，支持 URL 输入和文件上传。适配器需要处理封面 URL 或通过附件 API 上传。

### 10.3 试听音频和频谱图

均有对应的 `attachment.php?target=audition` 和 `attachment.php?target=spectrogram` iframe 上传接口。

### 10.4 无 IMDb / PT-Gen 字段

纯音乐站无视频相关字段（无 IMDb 链接、无 PT-Gen、无 MediaInfo）。

### 10.5 无视频编码/分辨率/音频编码字段

所有质量相关字段完全不同于视频站：
- 无 `codec_sel`（视频编码）
- 无 `standard_sel`（分辨率）
- 无 `audiocodec_sel`（音频编码）
- 使用 `standard`（音频格式）+ `medium`（物理媒介）+ `audio_mode`（整/分轨）

### 10.6 转载规则必填

`share_rule` 为必填字段，适配器需为转发资源选择合适的转载规则值。建议：
- 源站禁止转载 → 值=1
- 源站未明确 → 值=2（未经允许不得转载）
- 源站欢迎转载 → 值=3
- 不确定 → 值=255

### 10.7 候选状态

新发布的种子默认为候选状态（`status=pending`），需管理组/投票审核。如果发布者等级足够（Elite User+），可直接发布。
